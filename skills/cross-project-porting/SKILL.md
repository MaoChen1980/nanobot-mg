## When to Use

- 需要将复杂机制从源项目移植到目标项目时
- 遇到跨项目框架/库迁移的挑战时
- 发现源实现与目标环境存在架构差异时

## Steps

### 阶段一：解构源实现
1. 阅读源项目相关代码，理解机制的核心逻辑
2. 识别关键组件：数据模型、核心函数、依赖项
3. 梳理调用链路和状态流转
4. 提取可复用的模式和不变式

### 阶段二：并行探索目标结构
1. 探索目标项目的目录结构和代码组织
2. 识别目标项目中已有的相似机制
3. 对比源与目标的架构差异
4. 评估目标项目的约束和扩展点

### 阶段三：识别关键改动点
1. 列出源实现中需要在目标环境适配的部分
2. 评估每个改动点的复杂度和风险
3. 设计桥接方案或适配层
4. 制定移植优先级和迭代计划

## Verification

- [ ] 源实现的每个核心功能都能在目标项目找到对应实现
- [ ] 关键数据模型和接口已正确适配
- [ ] 移植后的机制通过基本功能测试
- [ ] 目标项目的编码规范和风格得到遵守

## Pitfalls

- **过度适配**：不要试图100%复制源实现，关注核心价值
- **忽略差异**：目标项目的约束可能与源不同，需灵活调整
- **一次性完成**：分阶段移植，每阶段验证后再继续
- **依赖遗漏**：确保所有必要依赖项已迁移

---

**Self-optimization**: 记录每次移植的差异点，形成项目特定的适配清单，加速后续类似任务。
