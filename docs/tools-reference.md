# 工具参考

NanoBot 的 AI Agent 可以调用以下工具来完成任务。工具分为几大类，按功能领域组织。

---

## 文件系统

| 工具名 | 说明 |
|--------|------|
| `read_file` | 读取文件内容。支持按行范围读取、SHA256 校验、section 模式（提取 Markdown 章节）。需在 `workspace` 目录内。 |
| `write_file` | 创建新文件或完全覆盖已有文件。自动创建父目录。支持后处理链：`then_grep`（验证）、`then_check`（类型检查）、`then_exec`（运行命令）。 |
| `edit_file` | 通过精确子串匹配或行范围替换来编辑已有文件。编辑前必须先调用 `read_file`（SHA256 验证）。 |
| `delete_file` | 删除单个文件。工作区保护 + 自动验证。具有危险操作检测。 |
| `move_file` | 移动或重命名单个文件。源和目标需在同一文件系统。默认启用危险检测。 |
| `grep` | 在工作区中搜索文件内容（正则表达式）。支持大小写开关、限制结果数、排除路径。 |
| `glob` | 在工作区中按 glob 模式搜索文件名。支持忽略路径。 |

## 搜索

| 工具名 | 说明 |
|--------|------|
| `grep` | 按正则表达式搜索文件内容。支持大小写、上下文行数、排除目录。 |
| `glob` | 按文件名模式搜索文件。 |
| `semantic_search` | 语义/向量搜索。查找概念和意图匹配，不依赖精确关键词。当你知道要找什么但不确定具体关键词/符号时使用。精确搜索请用 `grep`。 |
| `conversation_search` | 在对话历史中按字符子串匹配搜索（SQL LIKE）。用于查找过去对话中的具体内容。需要对话 ID 和时间范围。 |
| `memory_search` | 在记忆库中搜索。优先使用 FAISS 向量搜索，回退到 grep 关键词搜索。 |

## 代码分析

| 工具名 | 说明 |
|--------|------|
| `explore_module` | 获取代码文件或目录的结构化概览：类、函数、签名、文档字符串、行号、符号之间的交叉引用。目录视图显示按扩展名分组的文件、大小、子目录树。 |
| `scan_project` | 扫描项目目录，生成 `project_card.md`。自动检测语言、构建系统、测试框架、CI/CD 配置、入口点等。用于 Coding Agent 模式。 |
| `analyze` | 分析文本内容并返回结构化摘要：行数、词数、章节标题、按频率排序的关键词、关键词过滤行。支持从文件读取或直接传入文本。 |

## 执行

| 工具名 | 说明 |
|--------|------|
| `exec` | 执行 Shell 命令。安全校验（13 种危险模式检测、命令白名单、工作区边界检查）。支持超时控制、工作目录指定、后台异步执行。 |

## 网页

| 工具名 | 说明 |
|--------|------|
| `web_search` | 搜索互联网。支持 Brave、Tavily、DuckDuckGo、SearXNG、Jina、Kagi 提供商。返回标题、URL 和摘要。 |
| `web_fetch` | 获取 URL 内容并提取可读文本（Markdown 或纯文本）。支持正则过滤、截断预览、图片渲染。JS 密集型页面可能渲染不完全。 |

## 记忆

| 工具名 | 说明 |
|--------|------|
| `log_event` | 将带时间戳的事件追加到 `memory/events/{topic}.md`。事件形成不可变时间线，用于记录决策、用户偏好、Bug 修复、项目里程碑等。 |
| `memory_search` | 在记忆库中搜索。优先 FAISS 向量搜索，回退 grep。 |
| `semantic_search` | 在给定文本块或工作区文件中进行语义搜索。 |

## 自我诊断

| 工具名 | 说明 |
|--------|------|
| `assess_me` | 认知状态自评估。另一个 LLM 作为中立观察者阅读完整对话，报告方向是否正确、哪些假设需要验证、可能遗漏了什么。 |
| `debug_root_cause` | 根因分析。读取完整对话并从 20 种 RCA 方法中推荐结构化调查方案（分治、对比、回滚、假设检验、反向推理等）。 |
| `reframe` | 问题重述/降噪。提供事实（发生了什么、尝试了什么、出了什么问题）获得清晰、聚焦的回答，去除工具调用杂音。 |
| `self`/`config` | 运行时状态自检。检查 AgentLoop 的运行时配置：模型名、提供商、会话数、工作区路径等。 |
| `tool_call_log` | 查询工具调用历史。按会话、工具名、成功/失败状态、结果大小过滤。用于调试和追踪。 |
| `restart_agent` | 重启 Agent。当陷入循环、状态损坏或需要重置时使用。 |

## 子代理

| 工具名 | 说明 |
|--------|------|
| `spawn` | 生成一个或多个子代理。支持指定角色、指令、工具集等参数。子代理独立运行并返回结果。 |
| `tell_subagent` | Orchestrator 向子代理发送指令。子代理向主代理通信请用 `notify_orchestrator`。 |
| `notify_orchestrator` | 子代理向主代理发送非阻塞通知（fire-and-forget）。 |
| `cancel_subagent` | 取消正在运行的子代理任务。 |
| `check_subagent` | 检查子代理的运行状态、阶段、迭代次数、运行时间等。 |
| `list_subagents` | 列出所有运行中的子代理及其状态。 |

## My 工具

| 工具名 | 说明 |
|--------|------|
| `config` (my-tool) | 运行时状态检查与配置。查看和修改 AgentLoop 运行时变量（只读模式或可写模式）。 |

## 定时任务

| 工具名 | 说明 |
|--------|------|
| Cron 工具 | 管理定时任务。支持 `add`（创建）、`list`（列出）、`remove`（删除）、`update`（更新）、`test`（测试）五种操作。使用 cron 表达式或固定间隔调度。 |

详细说明请参见：[定时任务文档](cron.md)。

## MCP 工具

通过 MCP 协议连接的服务器提供的工具会自动注册到 NanoBot 中。每个 MCP 工具的名称格式为 `mcp_<server>_<tool>`。

详细说明请参见：[MCP 协议文档](mcp.md)。

## 消息

| 工具名 | 说明 |
|--------|------|
| `message` | 通过聊天通道向用户发送消息。用于异步通知、进度更新等。 |

## 检查点

| 工具名 | 说明 |
|--------|------|
| `save_checkpoint` | 保存当前工作状态为检查点。在风险操作前或完成里程碑后使用。包含文件快照和检查点描述。 |
| `list_checkpoints` | 浏览已保存的检查点，查看每个检查点的变更内容。 |
| `restore_checkpoint` | 回滚文件到之前保存的检查点状态。撤销误操作。 |
