## 脚本角色边界（v3.18 新增）

### 角色定义

| 组件 | 角色 | 职责边界 |
|------|------|---------|
| **SKILL.md** | 决策框架 + 行为规范 | 定义 Q-CAP 三问、四维评分逻辑、归因规则、禁止行为、输出格式规范 |
| **mga_full_analysis.py** | 数据获取工具 | 获取分钟数据 + V/OI + 简化四维评分；**不实现归因逻辑、席位解读、三问假设** |
| **mga_report** 输出字段 | 数据容器 | 存储脚本输出的原始数据；**不替代 skill 的分析结论** |

### 脚本与 Skill 的关系

**脚本负责（数据层）：**
- 调用 akshare 接口获取实时数据
- 计算 V/OI、涨跌%、四维评分（简化版）
- 席位数据降级标注（`SKIP(降级0.9)`）
- 数据哈希变化检测（跳过无变化推送）

**Skill Steps 负责（决策层）：**
- Step 0.5 归因层：三问假设 + 驱动因素分类 + 置信度系数
- Step 1 Q-CAP 三问：Q1 谁在打谁 / Q2 打在哪个价位 / Q3 谁能打赢（品种特定归因链）
- Step 4.5 Q-V 四问：验证归因是否被数据支持
- Step 4.6 反向分析：信号与价格背离时强制执行
- 席位数据解读：净多/净空方向判断（需数据可用时）

### 禁止行为（v3.18 新增）

**⚠️ 适用上下文**：以下禁止行为适用于「人类发起分析」场景（用户主动询问行情、主力意图等）。cron 自动化任务场景见下方「cron 场景执行边界」章节。

- ❌ **加载 skill → exec 脚本 → 用脚本输出替代 skill Steps**：`exec` 脚本拿到 `mga_report` 后，直接用脚本输出的 `操作建议` 列发送给用户，跳过 Step 1 Q-CAP 三问和 Step 0.5 归因层
- ❌ **脚本输出的操作建议（如"可持""不追"）作为最终操作建议**：脚本的 `get_action_recommendation()` 只是基于涨跌%+V/OI 的简单规则，不包含品种特定归因链、席位解读、多空实力对比
- ❌ **脚本跳过席位数据就声称"席位降级已标注"**：脚本只输出 `SKIP(降级0.9)` 字符串，但 Skill Steps 必须解读降级原因（CZCE/SHFE/DCE 席位接口在夜盘时段返回 0 行）并在归因层明确说明
- ❌ **只执行脚本的 V/OI 排行就声称"全市场扫描已执行"**：Skill Step 0.1 要求输出板块资金流向 + 品种活跃度判断，脚本只输出 V/OI 数值排行

### cron 场景执行边界（v3.19 新增）

**触发条件**：任务由 cron 自动触发（每 30 分钟定时执行），数据已由 `mga_full_analysis.py` 预取，目标是飞书通知。

**⚠️ 强制前置检查（必须在任何 exec 之前执行）：**

```
【cron 场景执行流程 — 必须严格按顺序执行】

Step 0: 读取 JSON 检查 _skipped 和 _skip_reason（第一优先级，必须同时读取两个字段）
   ↓
   ┌─────────────────────────────────────────────────────────────────┐
   │ _skipped == true 且 _skip_reason 含「数据未变化」等关键词       │
   │ → ✅ 允许 message("数据未变化，跳过本次分析")                  │
   │ → content = ""                                                  │
   │ → ⚠️ 禁止：exec 脚本 / 输出完整报告                           │
   └─────────────────────────────────────────────────────────────────┘
   ↓
   ┌─────────────────────────────────────────────────────────────────┐
   │ _skipped == true 且 assess_me 处于压制收敛期                     │
   │ → ❌ 禁止 message() — 零文字压制                              │
   │ → content = ""                                                  │
   └─────────────────────────────────────────────────────────────────┘
   ↓
    ┌─────────────────────────────────────────────────────────────────┐
    │ _skipped == false                                               │
    │ → 继续验证 5 项条件（见下方）                                  │
    │ → exec + message() 完成后 → ⚠️ 必须零文字 content = ""        │
    └─────────────────────────────────────────────────────────────────┘

**⚠️ Step 0 必须同时读取两个字段：**
- `_skipped` — 控制是否跳过执行
- `_skip_reason` — **必须读取以区分「数据未变化」和「assess 压制期」两个分支**
  - 含「数据未变化」「数据无变化」「hash 一致」→ 调用 message() 发简短说明
  - 含「assess 压制」「findings」「压制收敛」→ 零文字压制
  - 其他原因 → 按含「数据未变化」处理（保守策略）
- ❌ **禁止仅检查 _skipped == true 就决定 message() 调用**：必须先读取 _skip_reason 再判断
- ❌ **禁止跳过 _skip_reason 读取直接调用 message()**：论据不足，属于行为缺陷

Step 1: skill_search + read_file SKILL.md（第二优先级，_skipped=false 时必须执行）
Step 2: 验证 5 项条件
Step 3: exec + message（仅当 5 项条件全部满足时执行）
```

**⚠️ 时序约束：**
- **禁止先 exec 再检查 _skipped** — 这会导致静默循环（数据未变化时仍执行 exec + message）
- **检查顺序不可颠倒**：必须先判断 _skipped 状态，再决定是否执行后续流程

**cron 场景的 skill 执行路径**：

| 步骤 | 人类发起分析 | cron 自动化任务 |
|------|-------------|-----------------|
| Step 0 数据源盘点 | agent 实时调用 akshare | 脚本已预取，读取 JSON 缓存 |
| Step 0.1 全市场扫描 | agent 输出 V/OI 排行 | 脚本已输出，验证即可 |
| Step 0.5 归因层 | agent 执行 Q-CAP 三问 | **可简化为：脚本输出的归因标注已覆盖主要驱动因素时，可跳过手动归因** |
| Step 1-4 量价验证 | agent 逐品种分析 | 脚本已计算四维评分，验证数据一致性 |
| Step 5 决策输出 | agent 综合判断 | **可用脚本输出替代，但需满足以下条件** |

**cron 场景允许脚本替代 Skill Steps 的条件**：
1. `mga_full_analysis.py` 已成功执行（fees_info + akshare 日线已获取）
2. `mga_report` 包含六列完整数据（品种/方向/评分/置信度/席位方向/操作建议）
3. 席位方向列已由脚本正确标注降级原因（`SKIP(席位T-1不可用，降级×0.9)`）
4. JSON 中 `_skipped == false`（数据有变化，需发送通知）
5. **agent 已执行 skill_search + read_file SKILL.md**（确保 skill 加载链路完整）

**cron 场景禁止行为**：
- ❌ **禁止先 exec 再检查 _skipped** — 这是静默循环的根因，证据：JSON timestamp=05:08:06 显示 _skipped=true，但 agent 仍执行 exec(05:40:25) 和 message
- ❌ cron 任务跳过 skill_search 直接 exec 脚本（必须确保 skill 加载链路可追踪）
- ❌ 使用过期的缓存数据（`_skipped == true` 时不发送飞书，也不输出任何内容）
- ❌ 脚本输出席位方向为 `SKIP(降级0.9)` 时，agent 不验证降级原因就直接转发
- ❌ **输出脚本 Step 0 未输出的数据字段**：Step 0 全市场机会扫描仅输出 V/OI Top5（数值）、涨幅 Top3、跌幅 Top3；禁止自行合成 L1/L2/L3 逐层分数汇总或独立置信度字段
- ❌ **将燃料油 V/OI=1.60 标记为「⚠️极端踩踏，不追多」但脚本无 L3 信号支撑**：脚本 Step 0 输出无 L3 方向信号时，禁止输出「极端信号」判断
- ❌ **exec + message() 成功后输出状态摘要**：禁止在 cron 轮次中输出任何包含「已完成」「已发送」「强势/弱势品种汇总」「可持/做多品种」「本轮数据」等的状态确认文字，无论 assess_me 是否处于压制收敛期；**区分以下两种场景：**
  - `_skipped == true` 且 `_skip_reason` 含「数据未变化」→ 零文字 `content = ""`（符合 Rule 8.4）
  - **`_skipped == false` 且数据有实质变化（V/OI 跳升显著或逼近阈值）→ 必须输出有内容文字 + 飞书消息同步；禁止零文字输出（飞书消息与文字输出必须同步，语义断裂属于行为缺陷）**
  - **`assess_me 压制期（findings + 压制指令共存）→ 零文字压制（Rule 8.4 优先）**

### OUTPUT GATE 补充说明（v3.18 新增）

**⚠️ 适用上下文**：以下 OUTPUT GATE 要求适用于「人类发起分析」场景。cron 自动化任务场景见上方「cron 场景执行边界」章节，其中置信度和席位方向列可直接使用脚本输出。

**六列表格格式要求（必须全部包含）：**

```
品种 / 方向 / 四维评分 / 置信度 / 席位方向 / 操作建议
```

其中各列说明：

| 列名 | 来源 | 要求 |
|------|------|------|
| 品种 | 脚本 `mga_report` | 加 🌙 标注夜盘品种 |
| 方向 | 脚本 `涨跌` 列 | ↑/↓/- |
| 四维评分 | 脚本 `评分` 列（L1/L2/L3/L4） | 12分制，非百分比 |
| 置信度 | **Skill Steps 计算**（非脚本） | `总分/12 × 100% × 席位降级系数` |
| 席位方向 | **Skill Steps 解读**（非脚本） | 净多/净空/-（席位不可用时标注降级原因） |
| 操作建议 | **Skill Steps 综合判断**（非脚本） | 包含品种特定归因链 + 多空实力对比 + 仓位建议 |

⚠️ **席位方向列特殊说明**：
- 脚本 `mga_report` **不含席位方向列**（`席位方向` 列需 Skill Steps 解读席位数据后填充）
- 席位不可用时：输出 `-（席位T-1不可用，降级×0.9）` 而非空值
- 席位方向判断必须由 Skill Steps 执行，不能用脚本输出的 `SKIP(降级0.9)` 字符串替代

### ⚠️ 数据截断后的引用边界

**TRIGGER: 工具输出（`exec`、`grep`、其他）出现截断标记（`truncated: true` 或 `(truncated, N chars)`）**

**核心约束：摘要中的数值必须来自本轮可见的工具输出，禁止从历史记忆/working.md/缓存中补充数值。**

截断意味着你只能看到部分数据。摘要应：
- **只引用截断前输出中实际出现的数据**
- **不补充** 截断输出中未出现的数值（如涨跌幅具体数字、V/OI 比率等）

**如果截断的输出包含关键数据点但被截断：**
1. 用 `read_file` 读取 `result_file` 指向的完整输出文件
2. 从完整文件中提取所需数值
3. 摘要基于完整文件内容

**⚠️ 典型违规：**
- ❌ exec 输出显示「品种A ↑」但无具体涨跌幅 → 摘要写出「品种A +3.38%」（虚构）
- ❌ exec stdout 截断 → 摘要引用 working.md 中的历史数字
- ❌ 某指标数值不在 exec 输出中 → 摘要写出具体数字（虚构）

**正确模式：**
- ✅ 「强势品种：品种A ↑（方向来自 exec 输出，涨跌幅待从完整文件读取）」
- ✅ 仅引用截断前输出中实际出现的数据

**判断标准：摘要中的每一个数值，都必须能对应到本轮 tool 输出中的同一行或同一字段。说不出来源的数值 = 未准备好写入摘要。**

⚠️ **Step 0 全市场机会扫描数据引用规范（v3.20 新增）**：
脚本 Step 0 输出的数据范围如下，禁止输出范围外的数据：

| 脚本输出字段 | 允许引用 |
|------------|---------|
| V/OI Top5 | ✅ V/OI 数值（如燃料油 1.60） |
| 涨幅 Top3 | ✅ 涨幅数值和品种 |
| 跌幅 Top3 | ✅ 跌幅数值和品种 |
| L1/L2/L3 逐层分数汇总 | ❌ 脚本不输出此字段 |
| 独立置信度字段 | ❌ 脚本不输出此字段 |
| 席位方向信号（CZCE/SHFE/DCE） | ❌ 脚本不输出方向，仅标注降级状态 |

**典型违规示例**：
- ❌ 声称「燃料油 V/OI=1.60，L1+L2+L3=+5，置信度 31%」——后两项脚本无输出
- ❌ 声称「⚠️极端踩踏，不追多」但脚本 Step 0 输出 L3=+0——无信号支撑

**⚠️ 强制前置检查（每次分析前必须输出）：**

【Step 0 验证清单 — 本次分析已执行】
- [ ] Step 0.1 时间校验：`date` 命令已执行
- [ ] Step 0.2 数据源可用性：fees_info / akshare日线 / 席位数据
- [ ] Step 0.3 三问假设已形成：谁在打 / 打在哪个价位 / 谁能打赢（先执行 Q-CAP-0 品种分类再出三问）

【数据透明度声明】
- Layer 2.5 fees_info：[可用 / 最新价≈结算价 / 未执行]
- Layer 2 akshare日线：[昨收可用 / 数据冻结 / 未执行]

**禁止行为**：
- ❌ 未执行 date 命令就声称"时间校验已完成"
- ❌ 未获取 akshare 日线就声称"Layer 2 数据已确认"

- ❌ 未完成四维验证就声称"MGA 完整执行"
- ❌ 对 Layer 2.5 fees_info 数据使用 validate_ohlc()（无 OHLC 字段）
- ❌ 首次回复只覆盖持仓品种（JM/EB/CF/LC），未输出全市场 V/OI 排行和跌幅/涨幅排行（必须让用户一眼看到机会全貌，无需追问「别的品种呢」）
- ❌ 21:30 USDA / 02:30 夜盘收盘等关键时间节点用 fees_info 结算价判断方向（结算价≠实时价，15:59后为收盘价，21:00后仍为日盘收盘价，夜盘实时价需从新浪日线接口获取）
- ❌ 引用"快讯原文"描述外部冲击（Layer 1 快讯接口已全部废弃，详见数据时效分层规范；应基于 Layer 2.5 fees_info 数据驱动分析，或明确标注"外部推断/待验证"而非"快讯原文"）
- ❌ 捏造不存在的市场数据或新闻内容（外部推断/外部知识必须标注来源，不得伪装为数据原文；如引用外部信息形成归因假设，必须标注"推断"并说明"需结合实际数据验证"）
- ❌ 使用推断性归因语言（"外部冲击-地缘政治"、"外部XXX在主导"）—— 应改为观察性语言（如"据快讯整理"、"据市场观察"），归因标注只描述观察到的事实，不做因果推断
- ❌ **加载 skill 后跳过 Steps 直接 exec 脚本**：`skill_search` + `read_file` 后未按 Steps 执行 Q-CAP 三问、归因层、四维验证，直接 `exec` 脚本拿 `mga_report` 数据后发送给用户（详见「脚本角色边界」章节）
- ❌ **用脚本操作建议替代 skill 操作建议**：脚本 `get_action_recommendation()` 输出"可持/不追"后，直接作为最终操作建议发送给用户，跳过 Q-CAP 三问品种特定归因链

**归因表述正反示例表（Rule 1.5 规范）**：

> **核心原则**：归因必须基于**实际观察到的数据或事实**，不能基于**外部知识推断**。当使用外部推断时，必须标注"推断/待验证"。

| 场景 | ❌ 错误表述（推断性/捏造） | ✅ 正确表述（观察性/数据驱动） |
|------|--------------------------|------------------------------|
| **地缘事件归因** | 「伊朗五阶段报复打击，霍尔木兹不可通行」 | 「快讯提及巴林防空警报，霍尔木兹海峡油轮遭袭」 |
| **捏造快讯原文** | 「据快讯原文：霍尔木兹海峡关闭」 | 「据快讯整理：阿联酋油轮在霍尔木兹海峡遭袭」 |
| **行动代号归因（禁止）** | 「伊朗'纳斯尔2行动'已证实袭击油轮」 | 「霍尔木兹海峡南部油轮遭袭（阿联酋国防部确认）」 |
| **数据驱动** | 「受XX影响，价格上涨」 | 「XX品种+1.09%（数据驱动）」 |
| **外部推断** | 「市场预期美联储降息」 | 「CME FedWatch 显示降息概率XX%（数据驱动）」 |
| **无来源知识** | 「基本面显示供不应求」 | 「库存较上周降X%（数据驱动）」 |

**归因推断判断三步法（每步必须验证）**：
1. **原文关键词提取**：从快讯原文提取国家/地区、事件类型、行动主体（如有）
2. **映射检查**：输出文本中的每个归因词是否来自原文？若添加了原文无的词汇（如「刚宣布」「已证实」「Nasr-2」等）→ 禁止
3. **逻辑等价性**：若「威胁」后紧跟「已打击」「刚宣布」等词，且原文无对应行动声明 → 禁止

**禁止案例对照表**：

| 上下文仅有 | ❌ 错误推断（禁止） | ✅ 正确表述 |
|-----------|-------------------|-----------|
| 「威胁打击伊朗」 | 「美军已宣布发起新一轮打击」 | 「威胁已发出，后续行动待确认」 |
| 「油轮在霍尔木兹遭袭」 | 「伊朗Nasr-2行动已实施」 | 「霍尔木兹海峡油轮遭袭事件」 |
| 「巴林美军基地附近爆炸」 | 「美军打击伊朗目标」 | 「巴林美军基地附近爆炸（快讯原文）」 |

**判断规则**：
```
IF 归因内容超出了快讯原文或数据范围
  THEN 必须标注「推断/待验证」
  AND 不能使用「据XX报道」「据XX分析」等伪来源表述
  AND 归因置信度降低一档（高→中，中→低）
```

## Steps

### ⚠️ 强制输出前检查（每次分析前必须执行，禁止跳过）

**在输出任何评分、置信度或操作建议之前，必须完成以下检查清单：**

```
【Step OUTPUT GATE — 输出前强制验证】

□ Step 0 数据源盘点已完成（时间校验 / 数据源可用性已确认）
□ Step 0.1 全市场机会扫描已执行（V/OI排行 / 涨跌排行已输出）
□ 品种知识库已调用（JM/EB/CF/LC四品种分类已确认）
□ Step 0.5 归因层已完成（外部冲击 / 内生供需已标注）
□ Step 1 三问假设已形成（Q1/Q2/Q3 各品种均有输出）
□ L1 趋势评分已执行（MA20/MA40方向判断有结果）
□ L2 资金评分已执行（3日OI轨迹已分析）
□ L3 结构评分已执行（10日/20日高低点已判断）
□ L4 风控评分已执行（入场价已知 / 或标注"待定"）
□ 四维评分各维度得分已输出（L1/L2/L3/L4 各3分制）
□ 置信度已按公式计算（框架置信度分 = 四维总分/12×100% × 数据可用性系数）
□ 输出格式为六列表格（品种 / 方向 / 四维评分 / 置信度 / 席位方向 / 操作建议）

【禁止输出声明】
❌ 禁止在未执行L1/L2/L3评分的情况下输出任何"置信度"数值
❌ 禁止在未调用品种知识库的情况下输出三问结论
❌ 禁止用百分比替代12分制评分（如"置信度12%"不是有效输出）
❌ 禁止跳过席位数据分析（若席位不可用，必须标注降级原因）
❌ **禁止在六列表格中遗漏席位方向列**（脚本 `mga_report` 不含此列，必须由 Skill Steps 解读后填充；席位不可用时输出 `-（席位T-1不可用，降级×0.9）`）
```

### 核心执行流程（按顺序执行）

```
Step 0: 数据源盘点 + 全市场扫描（强制第一步）
  → 系统时间校验 → 并行探测所有数据源 → validate_ohlc()
  → futures_fees_info() 一次完成全品种成交额排名 + 板块资金流向 + OI 变化率 + 成交量健康度
  → 输出数据清单 + 本品种活跃度 + 板块背景冷暖

Step 0.1: 全市场机会扫描（强制，首次回复必须输出）
  → fees_info 获取全市场数据 → **按合约代码提取品种代码前缀 → 用品种代码列精确匹配 → 取持仓量最大合约**
    ⚠️ 匹配规则：必须用 `品种代码` 列（合约代码的大写前缀，如 `L`/`EG`/`SC`）精确匹配，不能用中文品种名匹配 `品种名称` 列
  → 首次回复必须包含以下三部分：
    ① **持仓品种快速结论**：所有持仓品种的 V/OI + 涨跌% + 席位方向（净多/净空/-）
    ② **全市场 V/OI 最高的前5品种**：V/OI > 0.8 的品种（强势信号排行）
    ③ **全市场跌幅/涨幅最大的前3品种**：关注机会
  → **全品种席位数据获取规范**：
    - CZCE：akshare.get_rank_table_czce(date='T-1') → 取成交量最大合约 → 计算净持仓=多头持仓-空头持仓
    - SHFE：akshare.get_shfe_rank_table(date='T-1') → 同上
    - DCE：akshare.get_dce_rank_table(date='T-1') → 超时降级（不重试，禁止反复拉）
    - 字段处理：席位DataFrame的持仓量字段是str（含逗号）→ 必须 `.str.replace(',','').astype(float)` 后再计算
    - ⚠️ 席位数据为 T-1 滞后，非实时
  → **夜盘品种标注**（所有输出中，对有夜盘的品种加 🌙 标记）：
    - SHFE全品种（CU/AL/ZN/NI/SN/PB/AG/AU/HG/BU/FU/RB/HC/SR/SS等）+ DCE工业品（I/JM/J/EB/PP/L/EG/LPG/PG等）+ CZCE主流（CF/SR/TA/MA/OI/RM/PF等）+ 农产品（DCE豆系/棕榈油等）
    - 标注位置：品种名称后加 🌙
  → **禁止行为**：只分析持仓品种，让用户追问「别的品种呢」才补全数据
  → **目的**：用户一眼看到机会全貌，无需追问

### ⚠️ 主动扩展全量概览规则

**TRIGGER: 用户提供持仓品种列表要求分析，但未明确限定"只看这些"**

**判断标准：**
- 用户提供了具体名单，但未明确说"只分析这几个"
- 用户关心的是全量机会，不是仅限列出的几个
- 可以从"别的呢"、"还有哪些"、"其他"等追问信号中识别

**ACTION:**
1. **初始回复即包含全量概览** — fees_info 一次性拉取全市场数据
2. **分层呈现** — 全量列出 V/OI 排行、涨跌排行，持仓品种放在重点分析区
3. **不要等用户追问才补充全量** — 默认意图是"看看全市场机会"

**禁止行为：**
- ❌ 只回复用户列出的持仓品种
- ❌ 用户追问"还有别的呢"才补充全量
- ❌ 认为用户没有明确说"全量"就不给全量

**正确模式：**
- ✅ 用户说"分析 A/B/C" → 初始回复即包含全市场概览，A/B/C 在重点分析区

Step 0.5: 归因层
  → 识别驱动因素：外部冲击 or 内生供需 → 标注归因置信度

**归因类型分类（必须穷举，选取最主导类型）：**

| 归因类型 | 子类 | 典型信号 | 归因置信度 |
|---------|------|---------|-----------|
| **外部冲击** | 宏观政策 | 央行加息/降息、汇率大幅波动、财政刺激 | 高（若价格传导链清晰） |
| **外部冲击** | 地缘政治 | 战争、制裁、航运中断、OPEC+会议 | 高（若品种直接相关） |
| **外部冲击** | 数据发布 | USDA报告、CPI/PPI、非农、库存数据 | 高（若在预期外） |
| **外部冲击** | 股市联动 | 期货跟随A股/美股/港股消息（股指→商品） | 中（需验证传导链和时间差） |
| **外部冲击** | 认知操纵 | **信息不对称**：借消息/时机操纵群体认知，引导资金和价格；信号：消息发布时机与价格异动高度同步 + 席位无对应开仓逻辑支撑 | 中（需交叉验证席位数据） |
| **内生供需** | 供给收缩 | 停产检修、天气灾害、矿山罢工 | 中（需验证实际影响） |
| **内生供需** | 需求变化 | 终端消费季节性、库存周期、补库/去库 | 中（滞后指标为主） |
| **内生供需** | 成本传导 | 原料涨价→下游跟涨（原油→化工品） | 中（传导有时滞） |
| **资金博弈** | 逼空/挤仓 | **放大机制**：持仓量极高+近月涨幅>>远月 / 席位极度集中（需有明确的放大对象） | 中（需事后验证，不可单独作为入场理由） |
| **资金博弈** | 多逼空 | 近月持仓>>可交割库存 / 散户被迫平仓（本质是流动性结构的极端化） | 中（尾声伴随大幅波动，方向常反转） |
| **无明确驱动** | 市场噪音 | 短期技术性波动、无消息配合 | 低（禁止强行归因） |

⚠️ **归因禁止：** 禁止无明确放大对象时将价格波动归因于"资金博弈"——这是描述性词语而非因果解释。资金博弈本质是放大已有信号（低库存/仓位集中），没有放大对象 = 没有归因。无法归因时标注「归因不确定」，不影响后续量价验证。

⚠️ **归因前置检查（洞见5）**：遇到突发事件导致的快速暴涨/暴跌时，先用「洞见5 情绪性逃跑识别表」判断是情绪性还是结构性。情绪性行情不归因于供需逻辑，只标注"情绪驱动"。

### 席位数据降级路径（归因层必须标注）

> **规则**：席位数据为0行时，必须在归因层标注降级原因。禁止跳过席位数据分析而不说明原因。

| 交易所 | 接口 | 超时/失败处理 | 置信度降级 |
|--------|------|--------------|-----------|

## 反向分析四问（信号与价格背离时执行）

> **核心理念**：市场永远不会错。归因和信号都指向看涨但价格不涨 → 一定有你没看见的力量在压制。找到它，比坚持你的归因更重要。

**触发条件**（任一）：
- 归因结论明确看涨，但价格在关键支撑位上方滞涨或阴跌
- 量价信号（OI+成交量）与价格方向背离超过 2 个交易日

**四问排查顺序**：

```
Q1: 谁在偷偷卖？
  → 检查仓单（注册仓单是否大增？产业在交货？）
  → 检查席位（多头集中度是否下降？空头是否有新资金入场？）
  → 检查 OI 结构（近月减仓远月增仓 = 移仓而非方向性交易）
  
Q2: 什么隐性压力没看到？
  → 社会库存 vs 显性库存（显性降=真降？还是转移到了隐性仓库？）
  → 进口到港/新产能（利好在 price in 了但供应在路上了？）
  → 替代品/相关品（品种涨了但替代品更便宜，需求被抢了？）
  → 汇率/政策（人民币升值/贬值在抵消价格变动？）
  
Q3: 我的归因是不是反了？
  → "利多"是否是滞后指标？（eg 库存下降是因供给收缩而非需求好）
  → "利多"是否已经 price in？（消息出来价格不涨 = 已经定价）
  → 我是否忽略了一个更重要的矛盾？（eg 看多 ER 但原油在跌）
  
Q4: 市场结构有没有问题？
  → 持仓量是否处于极端高位？（持仓极高+价格停滞 = 行情末端/拥挤交易）
  → 期限结构是否异常？（back 变 contango = 供需预期转变）
    - 获取方法：分别拉近月和远月合约日线，比较价格
    - 近月 > 远月 = Backwardation（现货溢价）；近月 < 远月 = Contango（期货溢价）
    - 结构转变比绝对水平更重要
  → 流动性是否枯竭？（成交量骤降 = 机器停了，方向不明）
```

**输出规则**：
```
反向分析结论：[压制力量已识别 / 未发现 / 分析可能错误]
修正动作：
  - 已识别压制力量 → 重新评估多空强弱，可减仓/放弃
  - 未发现隐藏力量 → 保留原判断但降低仓位（谨慎）
  - 分析可能错误 → 放弃原判断，重新做 Step 0.5 归因
```

## 底层哲学：最速曲线原理（第一性原理）

> **价格走阻力最小的路径，不是走你期望的方向。我们的任务是找到那条路，不是预测目的地。**

| 光学 | 市场 |
|------|------|
| 介质折射率 n | 市场阻力/不确定性 |
| 费马原理（光走时间最短路径） | 价格走阻力最小的路径 |
| 弯折处 = 介质突变带 | 弯折处 = 决定性资金的隐性底线/天花板 |

**三个推论**：
1. **不要抄底摸顶** — 底和顶是弯折带，是价格改变方向的地方
2. **等弯折完成再进入** — 不在弯折带里逆势，等价格进入直线段再顺势
3. **弯折带是观察点，不是操作点** — 真正的操作在直线段

**在 MGA 中的落地**：
- Q-CAP-4（决定性资金约束）= 找到折射率突变带
- "目标 vs 行动" = 判断当前是在弯折中（被动）还是在直线中（主动）
- 散户生存策略 = 只在直线段借动量，不在弯折带里逆向

## 核心理念

**先找因，后验证。** 价格/量/OI/席位都是结果，不是原因。必须先还原资金结构和归因，再用量价验证假设，而不是从量价出发反推原因。

**方法论的已知局限**：
- 资金结构层（Step 0）的判断依赖定性知识，无法精确量化
- 归因层和资金结构层的结论可能因信息不足而矛盾，应标注"归因不确定"而非强行统一
- 四维全通过不等于行情不会反转
- 任何置信度数字都是概率估计，不是预测

## 核心洞见 — 资金行为的第一性原理

以下洞见是 MGA 分析的底层逻辑。每个洞见对应一个资金行为规律 + 一个可操作的数据检测手段。**先理解洞见，再执行步骤。**

### 洞见1：资金在价格之前行动

> 成交量/持仓量的变化领先于价格变化。价格可以因为流动性枯竭而停留在原地，但资金已经在悄悄进出。等价格反映出来时，趋势已经走了一段。

**检测手段 — 量价背离信号表**：

| 价格行为 | 量/仓行为 | 资金在做什么 | 结论 |
|---------|----------|------------|------|
| 上涨 | 成交量递减 | 追高资金减少，上涨缺乏新买家 | 上涨乏力，不追多 |
| 下跌 | 成交量递减 | 杀跌资金减少，下跌缺乏新卖家 | 下跌乏力，不追空 |
| 上涨 | 持仓量递减 | 空头在平仓止损，不是新多入场 | 反弹而非反转 |
| 下跌 | 持仓量递减 | 多头在平仓止损，不是新空入场 | 回调而非反转 |
| 横盘 | 持仓量递增 | 多空在积聚，方向即将选择 | 等突破确认再入场 |
| 涨/跌 | 成交量骤增 + 持仓量骤降 | 大量仓位在平仓离场（踩踏），非方向性开仓 | 可能是流动性事件，不参与 |

**⚠️ 用法**：Step 2-4 四维验证时，先用此表判断量价关系是否健康，再进入品种具体分析。量价背离 = 四维验证不通过。

---

### 洞见2：资金永远在追逐确定性

> 资金不会在没有共识的地方浪费子弹。一个有真正确定性行情的品种，一定伴随全市场领先的成交额。成交额是市场用真金白银投票的结果，比任何分析师的判断都真实。

**检测手段 — 全市场扫描（Step 0.1 执行）**：
```
futures_fees_info() 获取所有品种最新行情
→ 成交额 = 最新价 × 成交量
→ 本品种在全市场中的排名百分位
→ 板块资金流向（按品种名称关键字分组）
```

| 成交额排名 | 活跃度 | 信号可信度 | 分析策略 |
|-----------|-------|-----------|---------|
| 前20% | 高活跃 | 高 — 资金集中，市场共识强 | 正常分析，信号可直接使用 |
| 20%-60% | 中活跃 | 中 — 有流动性但非焦点 | 信号需席位或 OI 二次确认 |
| 后40% | 低活跃 | 低 — 流动性不足，易操纵 | 不独立入场，仅作辅助参考 |

**⚠️ 一步到位**：只需调用一次 `futures_fees_info()` 即可完成全市场扫描。禁止为每个品种单独调 API。

---

### 洞见3：资金不消失，只轮动

> 资本永不休眠。当一个板块的确定性行情结束或拥挤度过高，资金必然流向下一个预期收益最高的板块。不是因为板块变了，而是因为资金的机会成本在变。一个品种的趋势强度永远受限于板块整体资金流向——资金流入板块 = 品种容错率高；资金流出板块 = 品种自身逻辑再好也难独善其身。

**检测手段 — 板块资金流向聚合**：
```
在 洞见2 的全市场扫描中一并计算：
→ 按品种名称关键字分组聚合板块成交额
→ 板块成交额占比 = 板块成交额 / 全市场总成交额
→ TOP3 资金流入板块 vs TOP3 资金流出板块
```

**输出格式**：
```
板块资金流向：
  [有色] 成交额占比 28%（+3%）↑ 资金流入
  [黑色] 成交额占比 22%（-2%）↓ 资金流出
  [化工] 成交额占比 15%（+1%）→ 持平
```

**应用规则**：本品种所在板块如果处于资金流出状态，即使品种自身信号好，也要标注"板块背景偏弱，需更严格验证"。反之亦然。

### 洞见4：极端博弈必然出清

> 期货是保证金交易，多空双方都在用杠杆。当持仓量处于极端水平，意味着大量资金在价格的两侧对赌。价格一旦向一方突破，亏损方的保证金压力会迫使减仓止损，止损指令本身就会推动价格继续同向移动——正反馈循环形成。所以持仓量极端从来不是稳定状态，它是趋势加速的前兆，不是趋势延续的确认。

**检测手段 — 持仓量异常检测**：
```
futures_zh_daily_sina(symbol="主力合约") 获取日线
→ 计算当日 OI 变化率 = (hold_today - hold_yesterday) / hold_yesterday
→ 判断：变化率 > 20% = OI 异常，需立即归因
→ 判断：OI 处于近期高位 = 多空博弈白热化
```

**⚠️ 注意事项**：
- `hold` 字段是累计口径，换月时会有跳空下降，换月当天的 OI 变化率不可信
- 检测前先确认当前合约是否在换月窗口（近月持仓向远月转移）
- 换月窗口的判断：近月合约持仓量连续下降 + 远月合约持仓量连续上升

### 洞见5：不确定性引发情绪性逃跑

> 资金追逐确定性（洞见2），所以资金也天然憎恶不确定性。当突发事件来临时——数据超预期、政策突变、地缘冲突——资金的第一反应不是"分析一下"，而是"先跑再说"。这种逃跑是情绪性的，不是结构性的。情绪性逃跑的特征是：**速度快、幅度大、但不可持续**。一旦不确定性消除或市场消化了消息，逃跑的资金会回来，价格也会修复。

**检测手段 — 情绪性波动识别**：

| 信号 | 情绪性逃跑（不可持续） | 结构性驱动（可持续） |
|------|---------------------|-------------------|
| 价格行为 | 瞬间暴跌/暴涨后快速反弹 | 稳步推进，不回补缺口 |
| 成交量 | 单根K线成交量异常放大后回落 | 成交量持续维持在较高水平 |
| 持仓量 | 暴跌时持仓量骤降（多头的平仓止损，非空头开仓） | 持仓量与价格同向持续变化 |
| 消息消化 | 消息出来30分钟后价格开始反转 | 消息出来后价格继续同向移动 |
| 跨品种 | 相关品种没有跟随 | 相关品种同步反应 |

**⚠️ 用法**：遇到突发事件导致的行情时，先用此表判断是情绪性还是结构性。情绪性行情不参与（或只做回归），结构性行情可参与趋势。**最危险的错误是把情绪性逃跑当作趋势开端的信号。**

### 洞见6：低成交量的价格波动没有意义

> 价格每变动一个 tick，都需要真实的资金去推动。成交量低 = 参与资金少 = 少量的订单就能把价格推到很远。这种价格移动不反映市场共识，只反映流动性不足。**不是所有价格变动都是信号。** 很多看起来漂亮的突破，在没有成交量的情况下，都是假的。

**检测手段 — 成交量有效性判断**：

```
在 洞见2 的全市场扫描中一并获取：
→ 品种成交量在全市场中的排名
→ 品种当日成交量 vs 该品种历史日均成交量

成交量健康度判断：
- 成交量排名前40% + 量价同向 = 信号可信
- 成交量排名后30% = 无论价格怎么走，都标注"低流动性行情"
- 当日成交量 < 历史日均成交量 50% = 流动性异常偏低，所有信号降档
```

**应用规则**：
- 低成交量环境下的突破 → 标注"注意低流动性"，等成交量确认再入场
- 高成交量环境下的突破 → 可信度高，可正常分析
- 成交量骤降（< 日均50%）+ 价格横盘 → 市场在等待，不是方向选择

## ⚠️ 数据时效分层规范

**核心原则：不同数据源的时间戳必须严格分离，禁止混用。**

```
Layer 1: 快讯新闻（akshare.futures_news_sina）
  - 来源：akshare.futures_news_sina()（新浪期货新闻，需实测验证）
  - 备用方案：akshare.futures_news_cninfo()（巨潮资讯期货新闻）
  - ⚠️ 东财快讯已废弃（qhweb.eastmoney.com/kuaixun 全部404，RPT_KUAIXUN_LIST 9501错误）
  - ⚠️ 快讯数据为推断性来源，必须标注「据快讯整理」而非「快讯原文」
  - 禁止：超出快讯原文范围做因果推断（如使用未经证实的行动代号"纳斯尔2行动"）
  - 置信度：0（数据不可用时，仅依赖定性判断）
  - 降级路径：若 akshare 快讯接口不可用 → 回退 Layer 2.5 fees_info 定性分析
```

Layer 2: akshare 昨日收盘价（futures_zh_daily_sina）
  - 来源：akshare futures_zh_daily_sina(symbol="CU2607")
  - 时间戳：前一交易日（如系统时间 2026-06-25 盘中时，数据为 06-24）
  - 用途：计算精确涨跌% + 持仓量变化 + ATR（High/Low数据支持14周期ATR计算）
  - ⚠️ 陷阱：系统时间≠数据时间，akshare 数据永远比"现在"晚1天
  - ATR数据源：akshare futures_zh_daily_sina 的 High/Low 字段 → 计算14周期 True Range → ATR=TR14指数移动平均（与标准ATR公式一致）
  - ⚠️ ATR≠实时价：ATR 是历史波动率指标，用于 L4风控 止损距离计算，非入场价参考
  - ⚠️ 具体计算步骤见下方「ATR计算步骤」章节
  - 置信度：中（历史数据，需结合 V/OI 异动判断今日是否加速）

Layer 2.5: akshare fees_info（futures_fees_info）
  - 字段：最新价 / 上日结算价 / 成交量 / 持仓量（⚠️无OHLC，无open/high/low字段）
  - ⚠️ 最新价行为（数据实测验证，2026-07-10 15:59:09 确认）：
    - **09:00-15:00（盘中交易中）**：最新价 = 实时成交价，波动剧烈
    - **15:59后（盘后）**：最新价 = 当日收盘价（**不是结算价**）
    - ⚠️ fees_info 无今日结算价字段；真正的结算价需从日线数据的 `settle` 字段获取
    - ⚠️ 正确验证方式：
      - 盘中对比：fees 最新价 vs 日线昨收（昨交易日收盘价）
      - 盘后对比：fees 最新价 vs 日线收盘价（两者应一致）
      - ⚠️ 禁止用 fees 最新价 vs fees 上日结算价 来判断涨跌方向（不同日期/口径，不可比）
  - V/OI 计算：盘中可用（成交量/持仓量字段存在）
  - 置信度：盘中时高（日盘有实时价），盘后时低（仅收盘价，无实时信号）

Layer 3:
  - 来源：akshare get_shfe_rank_table(date="20260624")
  - 时间戳：T-1 报告（席位数据比行情晚1天）
  - 用途：验证"谁在打"——多空席位集中度、净持仓方向
  - 置信度：中（滞后1天，但能穿透多空身份）
```

### validate_ohlc() 定义与使用规范

**函数签名**：`validate_ohlc(df: DataFrame) -> bool`
- 返回 `True`：数据有效（high >= low）
- 返回 `False`：数据无效（high < low，或任意字段为 NaN/0）

**内部逻辑**：
```
def validate_ohlc(df):
    if df.empty: return False
    if df['high'].min() < df['low'].max():  # 任意条 high < low
        return False
    if df[['open','high','low','close','volume','hold']].isnull().any():
        return False
    return True
```

**适用数据源**：
- ✅ akshare `futures_zh_daily_sina`：有 OHLC 字段 → 必须执行 validate_ohlc()
- ❌ akshare `futures_fees_info`：无 OHLC 字段 → 跳过 validate_ohlc()

**失败处理**：
- validate_ohlc() 返回 False → 该条数据标记为"无效"，尝试备用合约（同品种不同月份）
- 所有候选合约均无效 → 回退 Layer 2.5 fees_info（仅 V/OI，无 OHLC）
- ⚠️ 禁止：对 fees_info 数据调用 validate_ohlc()（会报错）

### ⚠️ 致命陷阱：昨收≠今行情
```
错误用法：
  akshare 沪铜 CU2607 昨收 = 103520
  → 误判："沪铜只跌了 -0.06%，表现抗跌"
  → 错！因为：夜盘+早盘累计跌幅远超 -0.06%，昨收已严重失真

正确用法：
  Layer 2（akshare昨收）：103520（-0.06%）→ 昨日收盘基本持平
  Layer 2.5（fees_info V/OI）：成交量/持仓量比平日放大 3 倍 → 盘中异动信号
  akshare日线今日缺口：跳空低开 → 今日在加速，非"抗跌"
  结论：昨收"抗跌"是假象，实际空头已在夜盘建仓
```

### ⚠️ 数据快照时间戳引用规范

**核心原则：引用具体品种数据时，必须用 `@时间戳` 后缀对齐到对应数据快照。**

```
问题场景：
  - 09:00 数据快照：锡席位列显示"多头席位净多"
  - 09:30 数据快照：锡席位列显示为空（DCE超时）
  - ❌ 错误引用："锡多头席位..."（未标注快照时间）
  - ❌ 混合快照："锡@09:00多头席位，但@09:30快讯..."（同一品种混合两个快照）

正确引用格式：
  锡@09:00 席位：净多XX手（多头席位集中）
  锡@09:30 快讯：+0.66%（上涨）

⚠️ 禁止行为：
  ❌ 引用某品种数据时不标注时间戳
  ❌ 同一品种混合来自不同快照的数据（除非明确说明时间差）
  ❌ 用09:00席位数据描述09:30的行情（席位数据是T-1滞后，不是盘中实时）
```

## 宏观定价品种验证规则（复盘 v3.24 新增）

> **触发条件**：品种属于外部定价型（AU/AG/SC原油/TA化工/有色金属）

### 规则1：宏观验证前置检查

**⚠️ 强制要求**：分析外部定价品种时，必须先验证宏观背景。

```
【宏观验证检查表】

□ 检查美元指数（DXY）方向：
  - 美元走强 → 黄金/白银/有色承压，宏观利空
  - 美元走弱 → 黄金/白银/有色受益，宏观利多

□ 检查原油价格方向（适用于TA/EG/MA/L/PP等化工品）：
  - 原油上涨 → 化工品成本支撑，宏观利多
  - 原油下跌 → 化工品成本松动，宏观利空

□ 检查实际利率/美联储政策（适用于AU/AG贵金属）：
  - 实际利率上升 → 黄金机会成本增加，宏观利空
  - 实际利率下降 → 黄金受益，宏观利多

□ 综合宏观判断：
  - 宏观利多 + OI信号看多 → 可做多
  - 宏观利空 + OI信号看多 → 降级为"观察"或删除
  - 宏观利多 + OI信号看空 → 降级为"观察"或删除
  - 宏观利空 + OI信号看空 → 可做空
```

### 规则2：宏观定价品种 OI 信号降权

```
IF 品种属于宏观定价型（AU/AG/SC/TA/EG/MA/L/PP/CU/AL等）
  THEN → OI 信号权重降至 0.5（非 1.0）
  AND → 宏观因素权重升至 0.5
  AND → 置信度计算：
    宏观验证通过（方向一致）→ 置信度维持
    宏观验证未通过（方向矛盾）→ 置信度降档
```

### 规则3：山脚分位宏观验证

```
IF 品种分位 < 25%（山脚）
  AND 品种属于宏观定价型
  THEN → 必须执行宏观验证前置检查
  AND → IF 宏观利空存在
    THEN → 分位低位优势被宏观抵消
    AND → 降级为"观察"或删除
    AND → 禁止输出"山脚安全边际"表述
```

### 规则4：OI 增加原因细分

```
【OI 信号来源判断】

IF OI 增加 + 价格下跌
  THEN → 空头减仓（反弹而非反转）→ OI信号不可信

IF OI 增加 + 价格下跌 + 宏观利空
  THEN → 空头减仓 + 宏观压制 → 禁止做多

IF OI 增加 + 价格下跌 + 宏观利多
  THEN → 可能多头建仓 → 需席位数据确认

IF OI 增加 + 价格上涨
  THEN → 多头建仓 → 信号可信（需席位验证方向）

IF OI 减少 + 价格下跌
  THEN → 多头减仓 → 信号可信

IF OI 减少 + 价格上涨
  THEN → 空头减仓（反弹而非反转）→ 禁止做多
```

### 规则5：内盘独立逻辑品种例外

> **核心原则**：当内盘有独立于宏观的驱动因素时（如中国囤油托底SC），宏观验证权重降低。

```
【内盘独立逻辑识别】

IF 品种有内盘独立驱动因素：
  - SC原油：中国炼厂趁低价囤油（需求侧采购）
  - TA PTA：聚酯工厂旺季备货（产业逻辑）
  - SN沪锡：下游逢低补库（低库存+产业备货）
  THEN → 内盘逻辑权重 > 宏观权重
  AND → OI 信号可维持原权重（1.0）
  AND → 但需标注"内盘独立逻辑，需持续验证"
```

**判断方法**：
1. 检查内盘供需是否有独立于外盘的变化（如库存变化、进口政策、季节性）
2. 检查内盘价格是否强于/弱于外盘（如SC强于WTI/布伦特）
3. 若内盘强于外盘 → 内盘逻辑主导 → 可维持 OI 信号权重

| 品种类型 | 代表品种 | OI权重 | 宏观权重 | 分析重点 |
|---------|---------|--------|---------|---------|
| 外部定价型-贵金属 | AU黄金/AG白银 | 0.5 | 0.5 | 美元指数+实际利率 |
| 外部定价型-原油 | SC原油 | 0.5 | 0.5 | OPEC+供给+中国需求 |

| 外部定价型-有色 | CU/AL/ZN/NI/SN | 0.5 | 0.5 | 美元指数+LME库存 |
| 内盘博弈型 | RB/HC/I/JM/J | 1.0 | 0.3 | 钢厂利润+政策预期 |
| 内盘博弈型-合金 | SF/SM | 1.0 | 0.3 | 钢厂招标+电力成本 |

### 规则7：禁止行为（复盘 v3.24 新增）

```
❌ 禁止对宏观定价品种（AU/AG/TA/SC等）仅凭OI信号做方向判断

❌ 禁止对内盘独立逻辑品种过度降级（如SC有中国囤油需求）

## Pitfalls

### ❌ 禁止 V/OI 分类边界错误（问题1修复）

**典型错误**：将 V/OI=1.44 归入「极端踩踏」（需 V/OI > 1.5），或未将 V/OI=1.6 归入「极端踩踏」。

**V/OI 阈值严格边界**：
| V/OI 区间 | 模式标注 | 操作含义 |
|---------|---------|---------|
| < 0.1 | 收敛观望 | 成交清淡，方向不明 |
| 0.1 ~ 0.5 | 正常 | 成交正常，无极端信号 |
| 0.5 ~ 0.8 | 踩踏 | 成交活跃，方向初现 |
| 0.8 ~ 1.5 | **强踩踏（盯紧）** | 踩踏确认，趋势强 |
| **> 1.5** | **极端踩踏（不追）** | 极端信号，禁止追方向 |

**判断规则**：
```
IF V/OI >= 0.8 AND V/OI <= 1.5
  THEN → 强踩踏（盯紧），操作：趋势确认后轻仓跟随
IF V/OI > 1.5
  THEN → 极端踩踏（不追），操作：禁止追方向，等踩踏结束
IF V/OI < 0.8
  THEN → 根据具体区间判断（收敛/正常/踩踏）
```

**禁止行为**：
- ❌ 将 V/OI=1.44 归入「极端踩踏」分类（应为「强踩踏」）
- ❌ 将 V/OI=1.6 归入「强踩踏」分类（应为「极端踩踏」）
- ❌ 笼统使用「高V/OI」而不区分具体区间

### ❌ 操作建议与 V/OI 极端矛盾（v3.23 新增，v3.25 重构）

**典型错误**：橡胶 V/OI=6.12>1.5（极端踩踏），但操作建议输出「可持」；PTA V/OI=3.23>1.5（极端踩踏），输出「观望」；焦煤 V/OI=3.45>1.5（极端踩踏），输出「可持」——与 V/OI 极端矛盾。

**操作建议生成规则（v3.25 重构）**：
```
IF V/OI > 1.5（极端踩踏）
  AND IF L1/L2 趋势方向与信号方向一致（如 L1=+1，L2=+1，多头信号）
    THEN → 操作建议 = "⚠️高V/OI盯紧，持仓可，控仓新仓不重仓"
    → 理由：高V/OI是成交拥挤的结果，趋势明确时持仓逻辑成立，但新仓须控仓
  ELSE IF V/OI > 1.5 且 L1/L2 趋势方向与信号方向背离（如锡 V/OI=5.03 但 L1=+1 L2=+3）
    THEN → 操作建议 = "⚠️极端踩踏，观望"
    → 理由：V/OI极高且趋势反转信号出现，须等踩踏结束

IF V/OI > 1.5 且无趋势信号支撑
  THEN → 操作建议 = "⚠️ 不追（极端踩踏）"
  AND → 禁止输出"可持"或"观望"

IF V/OI >= 0.8 AND V/OI <= 1.5（强踩踏）
  THEN → 操作建议 = "趋势确认后轻仓跟随" 或 "可持（强势）"
  AND → 禁止输出"不追"

IF V/OI < 0.8（正常/收敛）
  THEN → 操作建议 = "观望" 或 "可持"（视其他维度而定）
```

**07-14 复盘验证**：
- 原油 V/OI=3.93>1.5，地缘驱动明确（L1方向同向）→ 原建议"持仓可，不追" ✅ 正确
- 锡 V/OI=5.03>1.5，L1=+1 L2=+3 但输出"不追多" → 应区分趋势反转 vs 无趋势
- 燃料油 V/OI=3.93>1.5 → 原建议"持仓可，不追" ✅ 正确

**已修复品种（V/OI>1.5 → 修正为"⚠️ 不追"）**：
- 橡胶 RU：V/OI=6.12 → ⚠️ 不追（极端踩踏）
- 甲醇 MA：V/OI=5.11 → ⚠️ 不追（极端踩踏）
- 聚丙烯 PP：V/OI=4.27 → ⚠️ 不追（极端踩踏）
- 生猪 LH：V/OI=3.96 → ⚠️ 不追（极端踩踏）
- 20号胶 NR：V/OI=3.79 → ⚠️ 不追（极端踩踏）
- 焦煤 JM：V/OI=3.45 → ⚠️ 不追（极端踩踏）
- PTA TA：V/OI=3.23 → ⚠️ 不追（极端踩踏）

**禁止行为**：
- ❌ V/OI>1.5 时无条件输出"不追"（趋势明确时应区分"持仓可+控仓新仓"）
- ❌ 不参考 V/OI 阈值直接套用涨跌%生成操作建议
- ❌ 用"趋势确认"替代 V/OI 极端判断
- ❌ V/OI>1.5 且趋势反转时输出"持仓可"（矛盾）

### ❌ 禁止 MA 方向判断缺论据（问题2修复）

**典型错误**：锡 V/OI=3.19>1.5，下跌方向输出「⚠️空头踩踏，不追空」逻辑正确，但未引用 L1 评分作为论据。

**强制论据引用规则**：
```
IF 输出某品种的方向判断（如「看跌」「看涨」「空头踩踏」「多头踩踏」）
  THEN 必须引用该方向的 L1 评分数据作为论据
  AND 引用格式：「L1评分=[值]，MA20[>MA40/<MA40]→[方向]」
```

**正确示例**：
```
锡：V/OI=3.19>1.5，⚠️极端踩踏，不追
→ L1=-2（MA20<MA40，趋势向下），空头踩踏已确认
→ 禁止输出「空头踩踏」但不说明 MA 方向依据
```

**禁止行为**：
- ❌ 输出方向判断但不引用 L1 评分
- ❌ 输出方向判断但不引用具体 MA 关系（MA20 vs MA40）
- ❌ 用「趋势向下」等模糊表述替代 L1 评分数值引用

### ❌ 禁止地缘归因未 fetch 验证（问题3修复）

**典型错误**：输出「霍尔木兹→原油暴涨」「巴林防空警报」等地缘推断，但未展示 fetch 快讯原文步骤。

**地缘归因强制验证流程**：
```
Step 1: IF 归因类型为「外部冲击-地缘政治」
  THEN → 必须先执行 fetch 获取快讯原文或权威媒体确认
  AND → fetch 未成功 → 禁止输出具体地缘推断，只能标注「推断/待验证」

Step 2: fetch 成功后
  THEN → 引用快讯原文具体内容作为论据
  AND → 禁止超出快讯原文范围做因果推断

Step 3: IF 无法 fetch 验证
  THEN → 输出「地缘事件推断（待验证）」
  AND → 归因置信度降一档
```

**禁止行为**：
- ❌ 未 fetch 就输出「霍尔木兹→原油暴涨」等具体归因
- ❌ 用「据快讯整理」标注但未实际 fetch 验证
- ❌ 超出快讯原文范围做因果推断（如「霍尔木兹海峡关闭」→「原油暴涨」需 fetch 原文确认）

**正确示例**：
```
✅ 正确：「伊朗五阶段报复打击（快讯@14:32），原油+2.1%」
❌ 错误：「霍尔木兹不可通行→原油暴涨（未fetch验证）」
```

### ❌ 禁止席位数据状态描述歧义（问题4修复）

**典型错误**：用「全停」描述席位数据状态，但脚本输出「⚠️【席位数据陈旧警告】席位数据为 T-3」。

**席位数据状态区分规范**：
| 状态 | 定义 | 正确描述 |
|------|------|---------|
| **陈旧** | 数据存在但时效差（T-3） | 「席位数据陈旧（T-3）」 |
| **全停** | 完全无数据（接口返回空/超时且无法降级） | 「席位数据不可用（降级）」 |
| **可用** | 数据最新（T-1 或实时） | 「席位数据可用」 |

**禁止行为**：
- ❌ 将「陈旧」（T-3）描述为「全停」
- ❌ 将「可用」（T-1）描述为「陈旧」
- ❌ 笼统使用「数据异常」而不区分具体状态

**正确示例**：
```
席位数据：
- CZCE/SHFE：⚠️陈旧（T-3，09:30未更新）
- DCE：不可用（超时降级→日线定性）
```

### ❌ 禁止混合不同时间戳的数据快照

**典型错误案例**（assess_me 2026-07 实测）：
- 锡 V/OI=3.53 极端异常
- 09:30 锡席位列为空（DCE超时），但 agent 描述"多头席位"
- 根因：agent 混合了 09:00 快照的席位数据和 09:30 快照的快讯数据

**正确做法**：
1. 每次引用具体数据时，在品种名后加 `@时间` 标注快照来源
2. 同一品种不同维度的数据（席位/快讯/V/OI）必须来自同一快照，或明确标注时间差
3. 当某维度数据缺失（如席位为空）时，**禁止用其他快照的数据补充**，必须标注"席位@09:30暂不可用"

**判断规则**：
```
IF 引用品种X的数据
  THEN 在品种名后加 @快照时间（如锡@09:30）
  AND IF 引用多个维度的数据（席位/快讯/V/OI）
    THEN 确认各维度数据来自同一快照
    OR 明确说明时间差（如"席位@09:00，快讯@09:30"）
```

### ❌ fees_info V/OI 计算匹配规则

**典型错误**（assess_me 2026-08 实测）：
- 使用中文品种名（如"聚乙烯"）匹配 `品种名称` 列
- `name='聚乙烯'` → 无法匹配 `品种名称='线型低密度聚乙烯'`
- `name='聚乙烯'` → 无法匹配 `品种名称='聚乙烯月均价期货'`
- 结果：V/OI 全为 0.00，静默失败

**正确做法**：
1. 从主力合约代码提取品种代码前缀（如 `L2609` → `L`）
2. 用 `品种代码` 列（合约代码的大写前缀）精确匹配
3. 取持仓量最大的合约
4. 用该合约的 `成交量/持仓量` 计算 V/OI

**禁止行为**：
- ❌ 用中文品种名匹配 `品种名称` 列（包含品种全称/别名，无法精确匹配）
- ❌ 用字符串包含逻辑（`name in fee_name`）做模糊匹配
- ❌ 静默失败时不报错（`except: pass` 导致 voik 恒为 0.0）

## 三问穿透法

### Q1: 谁在打谁

**穿透什么**：多空双方的身份、动机、成本位

**操作顺序：先还原战场，再看 OI 数字**

> 分析一个品种时，必须先梳理「谁在打谁」——还原战场结构（参与者身份、动机、关系），然后才调用 OI/席位数据验证。不能倒过来先看 OI 数字再编故事。

## 品种分类体系

> 归因判断的第一步是分清品种属于哪一类。不同类别的品种，驱动逻辑完全不同。分析一个品种时，必须先梳理「谁在打谁」——还原战场结构（参与者身份、动机、关系），然后才调用 OI/席位数据验证。不能倒过来先看 OI 数字再编故事。

| 分类 | 定价权 | 核心驱动 | 代表品种 | MGA 分析重点 |
|------|--------|---------|---------|------------|
| **外部定价型** | 境外 | 外盘价格/境外供需/汇率 | 原油/化工链(TA/EG/MA/L/PP/EB)、有色(CU/AL/ZN/NI/SN/PB)、大豆/棕榈油 | 外盘联动 > 国内供需；外盘夜盘时段（北京时间凌晨）是关键观察窗口 |
| **内盘博弈型** | 国内 | 政策+国内供需+资金博弈 | 螺纹/热卷/焦煤/铁矿石/热卷/线材 | 政策扰动 > 供需现实；关注钢厂利润和房地产周期 |
| **混合型** | 内外各半 | 进出口政策+国内供需 | 橡胶(NR/RU)、棉花进口部分 | 进口政策 = 决定性变量；国内外价差是核心指标 |
| **政策市型** | 政策主导 | 收储/抛储/配额/补贴/宏观 | 棉花/白糖（国储+进口配额）、股指期货 | 政策节奏 > 市场供需；关注USDA/农业农村部报告 |
| **贵金属避险型** | 境外 | 美联储政策/美元指数/实际利率 | 黄金/白银（AG/AU） | 美元走势 = 核心驱动；避险情绪是反向指标 |
| **能源化工型** | 境外+国内 | 原油成本+国内供给侧 | 原油/燃料油/沥青/LPG/PG | 原油→化工成本传导；注意国内化工检修周期 |
| **农产品季节型** | 国内+境外 | 种植周期+进口政策+天气 | 玉米/豆粕/豆油/鸡蛋/生猪/苹果 | 季节性 > 供需；种植/上市周期是关键节点 |

**品种参与者结构速查表（显式化四方博弈关系）**：

| 品种类型 | 钢厂（下游买方） | 焦化厂/下游加工 | 矿山/供应商（卖方） | 贸易商（中间方） | 博弈特征 |
|---------|---------------|---------------|------------------|----------------|---------|
| **焦煤/焦炭** | 主导定价权（联合压价） | 夹心层（被双重挤压） | 挺价意愿强（成本支撑） | 进口煤是外生变量 | 钢厂 vs 煤矿 的价格战；焦化厂最弱 |
| **螺纹/热卷** | 主导定价权 | 建材加工 | 矿山挺价 | 贸易商蓄水池 | 终端需求主导；冬储博弈 |
| **铁矿** | 采购定价权弱（高度集中） | — | 四大矿山垄断 | 贸易商杠杆高 | 铁矿是卖方市场；钢厂利润决定需求 |
| **原油/化工** | 炼油厂（买方） | 化工加工 | OPEC+/中东 | 贸易商融资属性 | 原油成本推涨；下游被动跟涨 |
| **棉花/农产品** | 纺织厂（买方） | 纺纱/织造 | 农场/出口商 | 贸易商基差交易 | 政策托底 vs 产业做季节性空 |

> **使用规则**：Q1「谁在打谁」必须先定位本品种在上述表格中的博弈特征，再执行三问穿透法。不知道谁强谁弱，Q3 就是瞎猜。

**判断方法（Q-CAP-0，必须先于 Q1 执行）**：
```
IF 品种与原油/有色金属/美豆强相关（外盘定价）
  → 归因重点：外盘驱动 + 汇率 + 进出口政策
ELIF 品种是黑色系 + 国内主供（焦煤/螺纹/热卷/铁矿石）
  → 归因重点：国内政策（供给侧/环保/双碳）+ 钢厂利润 + 需求预期
ELIF 品种是铁合金（硅铁/锰硅）
  → 归因重点：钢厂招标价 + 电力成本 + 原料成本（兰炭/锰矿）
ELIF 品种有进口依赖 + 政策管制（白糖/橡胶/碳酸锂）
  → 归因重点：进口政策（配额/关税/国储）+ 国内外价差
ELIF 品种是养殖类（生猪/鸡蛋）
  → 归因重点：国内存栏周期 + 养殖利润 + 季节性消费 + 疫病
ELIF 品种是新能源类（碳酸锂/工业硅）
  → 归因重点：海外矿价 + 国内补贴政策 + 产能周期 + 下游去库
ELIF 品种是果蔬/软商品（苹果/红枣）
  → 归因重点：天气（花期/收割期）+ 产量预估 + 交割品级升贴水
ELIF 品种是航运类（欧线集运）
  → 归因重点：地缘冲突 + 运力供给 + 全球贸易 + 即期运价
ELIF 无法归入以上任何一类
  → 归因不确定，输出「品种分类待确认」，不影响后续量价验证
```

**⚠️ 常见错误**：把内盘博弈型品种（如焦煤）的下跌归因于"钢厂需求差"，但真实原因是"澳煤进口恢复压制国产焦煤价格"。判断错误会导致归因方向相反。

**反向检验（Q1 的自我否定步骤，必须执行）**：
> 完成 Q1 分析后，在调用任何数据验证之前，必须先问：
> **"如果我完全看反了，会看到什么信号？"**
> - 你认为是空头主力建仓 → 如果实际是多头减仓，信号是什么？
> - 你认为是产业套保 → 如果实际是投机空头，信号是什么？
>
> 如果反向信号已经存在或正在出现，**暂停 Q2/Q3，先回到 Q1 重做**。

### Q2: 打在哪个价位

**穿透什么**：关键支撑/阻力在哪里，多空的成本位和目标位

| 价位类型 | 定义 | 意义 |
|---------|------|------|
| 结构支撑 | 10日/20日/40日低点 | 多头防线 |
| 结构阻力 | 10日/20日/40日高点 | 空头防线 |
| 整数关口 | 15000/14000/10000等 | 心理位置 |
| 持仓密集区 | OI堆积价位 | 止损密集区 |

**关键判断**：价格跌破支撑后，如果 OI 不增，说明空头没认真打，可能是假跌破。
**加速突破判断**：akshare日线跳空缺口 > 昨收幅度 × 3倍，或 fees_info成交量/持仓量异动 → 行情正在加速 → 不抄底

### Q3: 谁能打赢

| 实力指标 | 强空头 | 强多头 |
|---------|-------|-------|
| 席位 | 净空大 + 席位集中 | 净多大 + 席位集中 |
| OI | 持续增仓 | 持续增仓 |
| 仓单 | 仓单增（产业卖保） | 仓单减（产业买保） |
| 基差 | 贴水加深 | 升水扩大 |
| 价格加速 | 盘中跌幅 >> 昨收跌幅 | 盘中涨幅 >> 昨收涨幅 |

⚠️ 持仓量下降 + 价格加速 → 空头陷阱或流动性踩踏

## 夜盘品种分类体系

期货市场夜盘时段（21:00 - 次日 02:30）是博弈最活跃时段，不同品种夜盘参与程度差异显著：

| 夜盘档位 | 交易时段 | 代表品种 | 博弈特征 |

| **无夜盘** | 日盘仅 09:00-15:00 | 粳稻/粳米/胶合板/纤维板/早籼稻/油菜籽/花生仁（部分农产品） | 国内独立定价，政策主导 |
| **部分夜盘** | 夜盘 21:00-23:00（日盘 09:00-15:00） | 部分化工/农产品 | 23:00 休盘后跟随外盘，次日补涨/补跌 |

```

   → 全夜盘，21:00-02:30
ELIF 品种属于 DCE 工业品（I/JM/J/EB/PP/L/EG/LPG/PG/M/Y/P等）

ELIF 品种属于 CFFEX 股指（IF/IC/IM/IH）
   → 无夜盘（股指期货无夜盘）
ELIF 品种属于农产品（苹果/红枣/粳稻/早籼稻等）
   → 无夜盘

   → 部分夜盘，参考具体品种合约

**⚠️ fees_info 数据在夜盘时段的局限性**：
- 21:00 后 `futures_fees_info` 的「最新价」= 日盘收盘结算价，非夜盘实时价

## 四维确认框架

四维是验证三问的数据工具，不是目的。

**⚠️ L4 特殊约束：L4 依赖入场价，不可在入场前评分。**
- L1/L2/L3：在入场信号出现后进行验证（信号确认工具）
- L4：需入场价确认后方可验证止损可行性
- 若入场价未知：L4 标记为"待定"，入场价确认后补全

**⚠️ L1/L2/L3 数据获取前提（必须先执行 Step 0 数据源探测）：**
- L2 OI轨迹 → akshare `futures_zh_daily_sina(symbol)` 的 `hold` 字段
  ⚠️ 实测结论（CU2607，239日数据）：
  - `hold` 不是市场总OI（实测最大527手 vs CZCE席位数据576,630手，差3-4个数量级）
  - `hold` 行为：单调递增（随时间累积）+ 换月跳空下降，比值 hold/volume 从1.0增至21.0
  - 正确用途：观察单个合约自身 hold 变化趋势（增/减），不可用于估算市场份额或计算 V/OI 比
  - V/OI 比应改用 `futures_fees_info` 的「成交量/持仓量」字段
- L3 10/20日高低点 → akshare `futures_zh_daily_sina(symbol)` 的 `high/low` 字段，取10日/20日最高最低
- L4 ATR → 同上 High/Low 字段，详见下方「ATR计算步骤」

| 维度 | 数据来源 | 验证条件 |
|------|---------|---------|
| **趋势面** | MA20/MA40方向 | 价格在MA40同侧，MA20在MA40同向即可确认趋势 |
| **资金面** | 3日OI轨迹 + 成交额排名 | OI同向且成交额活跃（前60%）表明资金支持，低活跃品种OI信号需谨慎 |
| **结构面** | 10日/20日高低点 | 突破/跌破前高前低确认结构 |
| **风控面** | ATR止损 | 止损距离 ≤ 1.0×ATR（入场价已知时） |

### ATR计算步骤（风控面 必备前提）

akshare `futures_zh_daily_sina` 无预置 ATR 字段，需自行计算：

```
Step 1: 获取历史 High/Low/Close 数据
  df = akshare.futures_zh_daily_sina(symbol="JM2609")
  取最近 20+ 条日线（计算14日ATR需要至少15条历史数据）

Step 2: 计算 True Range（每一条）
  TR = max(H - L, |H - Close_prev|, |L - Close_prev|)

Step 3: 计算 14 日 ATR（标准 Wilder's smoothing，即 EMA 等效）
  ATR_14 = EMA(TR 序列, period=14)  # pandas.ewm(alpha=1/14) 或 talib.ATR()
  注意：与前一日 ATR 的递推关系为 ATR_new = ATR_prev × 13/14 + TR_latest / 14

Step 4: 计算止损距离（用于风控面验证）
  止损距离 = |入场价 - 参考ATR支撑位| / ATR_14

  参考ATR支撑位：取最近 5 条 Low 的最小值（多头止损）
                  或最近 5 条 High 的最大值（空头止损）
```

⚠️ **不足14日数据时的处理规则**：
- 数据 ≥ 10 条：可用 ATR_10（10日均值），标注"数据不足，ATR压缩"
- 数据 < 10 条：**风控面验证不通过**，禁止用其他指标替代
- 禁止发明新指标替代 ATR（如用日波幅替代）

### N值波动率决策规则

| 品种ATR历史均值（14日ATR/收盘价）| 参考N值 | 说明 |
|------------------------------|---------|------|
| 波动率 > 3.0% | N = 3.0 | 高波动品种（原油、碳酸锂、某些农产品） |
| 波动率 1.5% ~ 3.0% | N = 2.0 | 中等波动（大多数工业品：铜/铝/螺纹/铁矿石） |
| 波动率 < 1.5% | N = 1.5 | 低波动品种（黄金/白银等贵金属） |

**计算方法**：在 Step 4 得到 ATR_14 后，计算 `ATR_14 / Close_latest × 100%` 即为波动率
**操作含义**：N 值越大 → 允许止损空间越大 → 信号要求更严格

### V/OI 速度阈值表

V/OI = 成交量 / 持仓量，衡量成交活跃程度与方向力度。

| V/OI区间 | 模式标注 | 操作含义 |
|---------|---------|---------|
| < 0.1 | 收敛观望 | 成交清淡，方向不明 |
| 0.1 ~ 0.5 | 正常 | 成交正常，无极端信号 |
| 0.5 ~ 0.8 | 踩踏 | 成交活跃，方向初现 |
| 0.8 ~ 1.5 | 强踩踏 | 踩踏确认，趋势强 |
| > 1.5 | 极端踩踏 | 极端信号，不追 |

## 决策规则

### 入场规则
```
IF 三问假设已形成
AND 四维验证通过（趋势/资金/结构/风控四维度均确认）
AND 有明确的止损位
AND 席位（若可用则验证，不可用则降档后入场）
AND 已确认各数据源的时间戳（昨收/盘中/席位T-1）
THEN → 入场
```

### 止损规则

| 信号 | 操作 |
|------|------|
| 价格反向突破结构位 | 止损 |
| OI突然大降（>10%）| 先减仓，确认后再处理 |
| 新信号反向（同一品种）| 先减仓，不反手 |
| 持仓量降 + 盘中加速（日线跳空>>昨收×3 或成交量异常放大）| 谨慎，可能是流动性踩踏 |

### 散户生存策略（所有决策输出的必填项）

| 维度 | 规则 |
|------|------|
| **决定性资金当前位置** | [安全区]→赔率好可持仓过夜；[中性区]→只能日内止损要小；[危险区]→不追 |
| **开仓时机** | 等大资金先动，我们再动（不领先） |
| **平仓时机** | 大资金未平，我们先平（不最后） |
| **持仓时限** | 安全区→1-2天；中性区→当日必须平；危险区→不开仓 |
| **仓位** | 试探仓≤10%，确认后加到≤30%；危险区不加仓 |

### 席位数据结构处理规范

**CZCE（get_rank_table_czce）和 SHFE（get_shfe_rank_table）返回格式**：
```
返回类型：dict{合约代码: DataFrame}
  - 字段：rank / vol_party_name / vol / vol_chg / long_party_name / long_open_interest / long_open_interest_chg / short_party_name / short_open_interest / short_open_interest_chg / symbol / variety
  - ⚠️ 陷阱1：「成交量」和「持仓量」字段是 str 类型，含逗号（如"120,516"）
    → 必须用 `.str.replace(',','').astype(float)` 转换后再计算
  - ⚠️ 陷阱2：返回结果中**无预置净持仓列**，需自行计算
    → `净持仓 = long_open_interest - short_open_interest`
  - ⚠️ 陷阱3：若直接访问 `df['净持仓列名']` 会触发 KeyError
    → 先用 `list(df.columns)` 确认实际列名，再访问
```

**DCE（get_dce_rank_table）返回格式**：
```
返回类型：dict{合约代码: DataFrame}（结构同CZCE/SHFE）
⚠️ 陷阱：DCE 接口极易超时（>30s），超时后直接判定不可用
→ 禁止反复重试 → 降级至0.3置信度
```

### DCE品种降级路径

DCE品种（铁矿石/焦煤/焦炭/塑料/豆粕等）席位接口全灭时：
```
Step 1: 确认DCE席位不可用（DCE超时→直接降级0.3，禁止重试）
Step 2: ⚠️ 东财快讯已废弃 → 直接进入 Step 3
Step 3: akshare.futures_zh_daily_sina(symbol="主力合约") 获取商品日线OHLC
         → 若合约不存在（空列表）→ 回退 Layer 2.5 fees_info，仅用 V/OI + 涨跌方向定性
Step 4: 综合信号给出定性归因结论
Step 5: 置信度：仅OI确认→0.7；仅结构位确认→0.5；定性归因辅助→0.4
```

**⚠️ 降级禁止输出精确价位（新增）**：
席位数据降级后，分析结论仅供定性参考，严禁输出精确价位。具体禁止项：
- ❌ **禁止输出精确止损位**（如「止损MA5」「止损1420」「止损ATR×1.5」等）
- ❌ **禁止输出精确目标价**（如「目标1550」「压力位1500」等）
- ❌ **禁止输出精确入场价建议**
- ✅ **必须标注**：「仅定性参考，不构成投资建议」
- ✅ **操作建议仅输出方向和仓位建议**（如「方向偏多 / 仓位≤10% / 等待席位数据恢复后补充风控」）
- ⚠️ **判断标准**：如果输出的是「价位」或「价格数字」，在席位降级状态下必须省略或标注「待定」

**⚠️ 降级后操作建议必须明确表述（禁止含糊「观望」）**：
- 降级后输出操作建议时，必须明确说明「止损/目标价需结合实时席位数据确认」，不得仅以「观望」「轻仓观望」等模糊表述替代
- ✅ 正确示例：「方向偏多，仓位≤10%；止损/目标价需结合实时席位数据确认后给出」
- ❌ 错误示例：「观望」（未说明后续如何给出具体建议，用户无法判断是放弃分析还是暂时等待）
- **根因**：降级≠放弃分析，明确告知用户分析框架完整但数据源降级，需要席位数据补充风控参数
