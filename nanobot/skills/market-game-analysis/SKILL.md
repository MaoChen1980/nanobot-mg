# MGA — Market Game Analysis

## 核心理念

**先找因，后验证。**

价格/量/OI/席位都是结果，不是原因。MGA的目标是先还原资金结构（谁的钱、为什么来）和归因（驱动因素是什么），再用量价/OI/席位来**验证**假设，而不是从量价出发反推原因。

**⚠️ 方法论的已知局限**：
- 资金结构层（Step 0）的判断依赖定性知识，无法精确量化——"谁的钱"是概率判断，不是事实断言
- 归因层（Step 0.5）和资金结构层的结论可能因信息不足而互相矛盾，此时应标注"归因不确定"而非强行统一
- 量价验证层（四维）永远落后于实时行情——四维全通过不等于行情不会反转
- 任何置信度数字都是概率估计，不是预测——70%置信度意味着30%概率方向看反

**判断流程（从因到果）**：
1. **资金结构层**（先于一切）— 谁的钱、他们的行为逻辑、宏观背景
2. **归因层** — 驱动是外部冲击还是内生供需
3. **三问假设** — 基于资金结构和归因推导，不是基于量价快照
4. **量价验证** — 席位/OI/仓单/基差/MA20/MA40 验证假设，不做假设来源

### ⚠️ 致命陷阱：昨收≠今行情（2026-06-25 实测验证）

```
错误用法：
  akshare 沪铜 CU2607 昨收 = 103520
  → 误判："沪铜只跌了 -0.06%，表现抗跌"
  → 错！因为：东财快讯显示沪铜夜盘跳空 -1.28%，早盘跌超1%

  Layer 2（akshare昨收）：103520（-0.06%）→ 昨日收盘基本持平

**validate_ohlc() — OHLC数据校验函数（每次获取行情数据后必执行）：**
```python
def validate_ohlc(df: pd.DataFrame) -> pd.DataFrame:
    """剔除 high < low / open <= 0 等 OHLC 不变量异常，防止脏数据进入分析"""
    invalid = (
        (df["high"] < df["low"]) |
        (df["high"] < df["open"]) | (df["high"] < df["close"]) |
        (df["low"] > df["open"]) | (df["low"] > df["close"]) |
        (df["open"] <= 0) | (df["high"] <= 0) | (df["low"] <= 0) | (df["close"] <= 0)
    )
    n_invalid = invalid.sum()
    if n_invalid > 0:
        print(f"[WARN] validate_ohlc: 剔除 {n_invalid} 行脏数据")
    return df[~invalid]
```
- 调用时机：akshare / mootdx / baostock 任一接口获取日线后，立即执行
- 常见异常：`high < low`（数据错误）、`open <= 0`（停牌/未上市）、`close=0`（接口返回0值）

│  - 两者是不同变量，指标计算时不可混用                │
│                                                     │
│  【理论定义】                                       │
│  - 正常市场下，Volume变化率dV/dt近守恒             │
│  - 理论支撑：Kyle(1985)、Glosten-Milgrom(1985)     │
│    市场微观结构模型 — 参与者结构短期稳定             │
│  - 可计算指标：dV/dt（成交量变化率）/ Bid-Ask Spread │
│  - dL/dt ≠ 0 → 市场结构变化预警，非"守恒打破"    │
│                                                     │
│  【当前指标的修正（重要）】                         │
│  ⚠️ 原Step 0.2偏差系数公式 "|涨跌%|×10/OI" 存在   │
│  概念错误：                                        │
│  - OI≠Volume，OI是持仓量不是成交量                 │
│  - 该公式实际描述的是"持仓规模约束下的价格弹性"    │
│  - 不是"流动性守恒"的验证                          │
│  - 需改用 Volume（成交量）数据重新计算              │
│                                                     │
│  【联合信号监测（需Volume+OI联合数据）】            │
│  - Volume放大 + OI不增 + 价格涨 → 空头踩踏        │
│  - Volume放大 + OI同增 + 价格涨 → 多头真建仓      │
│  - Volume缩小 + OI增大 → 散户离场，机构暗中建仓   │
│                                                     │
│  【输出格式】                                       │
│  - 相空间模式：[类型编号+名称]                      │
│  - 趋势强度：[强/中/弱/待确认]                     │
│  - 操作提示：[基于模式的初步提示，非最终决策]       │
│                                                     │
│  → 进入 Step 0.5（归因层）                         │
└─────────────────────────────────────────────────────┘

## 三问穿透法（详细版）

### Q1: 谁在打谁

**穿透什么**：多空双方的身份、动机、成本位

**⚠️ 操作顺序：先还原战场，再看 OI 数字**
> 分析一个品种时，必须先梳理「谁在打谁」——还原战场结构（参与者身份、动机、关系），然后才调用 OI/席位数据验证。不能倒过来先看 OI 数字再编故事。

**品种参与者结构速查表**：

| 品种类型 | 主要参与者 | 决定性力量 | 典型动机 |
|---------|-----------|-----------|---------|
| **原油/能化** | OPEC+产油国、俄罗斯、美国SPR、炼厂、贸易商（托克/维多）、对冲基金 | OPEC+供给调控 | 控产挺价/抢占份额 |
| **农产品** | USDA、四大粮商（ABCD）、农场主、主权基金、投机基金 | USDA报告+天气 | 预期管理/季节性套保 |
| **有色金属** | 矿山（必和必拓/自由港）、冶炼厂、贸易商（五矿/嘉能可）、LME基金 | 矿山供给+中国需求 | 成本支撑/利润锁定 |
| **黑色系** | 钢厂、铁矿石供应商（淡水河谷/力拓/必和必拓）、焦化厂、贸易商 | 钢厂定价权+矿山成本 | 利润锁定/成本传导 |
| **贵金属** | 央行（美联储/欧洲央行）、ETF管理机构（金条银行）、矿产商、投机基金 | 央行政策+实际利率 | 避险/抗通胀/投机 |
| **软商品** | 主产国政府、出口商、进口国政府、投机基金 | 主产国供给+汇率 | 贸易平衡/季节性 |

**战场还原三问**（先于 OI 数据调用）：
1. **这个品种的定价权在谁手里？** （如原油看 OPEC+，铜看中国需求）
2. **当前谁是潜在的空头/多头？** （产业套保？投机资金？央行？）
3. **他们的成本位/目标位在哪里？** （如矿山cash cost是铁矿的底部支撑）

| 信号 | 穿透结论 |
|------|---------|
| 席位净空 + OI增 | 空头主力建仓，席位数据验证 |
| 席位净多 + OI减 | 多头止盈，非空头打压 |
| 仓单增 + 价格跌 | 产业套保卖盘，压制价格 |
| 仓单减 + 价格涨 | 产业买保，支撑价格 |
| **持仓量骤降 + 价格加速下跌** | 多头踩踏/空头平仓，非基本面驱动，需谨慎 |

**席位数据优先级**（席位是验证"谁"的核心）：
- 净空集中度高 → 空头主力明确
- 净多集中度高 → 多头主力明确
- 席位分散 → 散户行情，非主力行为

**⚠️ 反向检验（Q1 的自我否定步骤，必须执行）**：
> 完成上述 Q1 分析后，在调用任何数据验证之前，必须先做反向检验：
>
> **"如果我完全看反了，会看到什么信号？"**
>
> - 你认为是空头主力建仓 → 如果实际是多头减仓，信号是什么？（席位净空 + OI减）
> - 你认为是产业套保 → 如果实际是投机空头，信号是什么？（席位集中度低 + OI增）
> - 你认为是宏观基金做空 → 如果实际是央行干预，信号是什么？（成交量异常放大 + 无明显席位集中）
>
> **操作规则**：如果反向信号已经存在或正在出现，**暂停Q2/Q3，先回到Q1重做**。不得以"已经形成假设"为由跳过反向检验。

### Q2: 打在哪个价位

**穿透什么**：关键支撑/阻力在哪里，多空的成本位和目标位

**⚠️ 盘中价位判断必须区分"昨收价位"和"实时价位"：**

| 价位类型 | 定义 | 意义 |
|---------|------|------|
| 结构支撑 | 10日/20日/40日低点 | 多头防线 |
| 结构阻力 | 10日/20日/40日高点 | 空头防线 |
| 整数关口 | 15000/14000/10000等 | 心理位置 |
| 前高/前低 | 历史极值 | 趋势确认位 |
| 持仓密集区 | OI堆积价位 | 止损密集区 |

**关键判断**：价格跌破支撑后，如果OI不增，说明空头没认真打，可能是假跌破。

**加速突破判断**：快讯盘中跌幅 > 昨收跌幅 × 3倍 → 行情正在加速 → 不抄底

### Q3: 谁能打赢

**穿透什么**：多空双方实力对比

| 实力指标 | 强空头 | 强多头 |
|---------|-------|-------|
| 席位 | 净空大 + 席位集中 | 净多大 + 席位集中 |
| OI | 持续增仓 | 持续增仓 |
| 仓单 | 仓单增（产业卖保） | 仓单减（产业买保） |
| 基差 | 贴水加深 | 升水扩大 |
| 价格（加速信号）| 盘中跌幅 >> 昨收跌幅 | 盘中涨幅 >> 昨收涨幅 |

**⚠️ 持仓量下降 + 价格加速 → 空头陷阱或流动性踩踏**
（PVC持仓降-4.4% + 价格涨1.04% = 空头减仓反弹，非真涨）

## 四维确认框架

四维是验证三问的数据工具，不是目的。

| 维度 | 数据来源 | 通过条件 |
|------|---------|---------|
| **L1趋势** | MA20/MA40方向（注：商品期货可调整为10/20日线，MA20在MA40同侧 + 价格在MA40同向） | 价格在MA40同侧，MA20在MA40同向 |
| **L2资金** | 3日OI轨迹 | OI和价格同向变化（⚠️ 注意：akshare `hold`=期货公司保证金口径，≠ 交易所持仓量OI，混用会导致判断错误） |
| **L3结构** | 10日/20日高低点 | 突破/跌破前高前低 |
| **L4风控** | ATR止损 | 止损=入场价±N×ATR（N由品种特性决定，参考值1.5） |

**四维评分计算规则**：

每维满分 3 分，总分 12 分。

| 维度 | 通过条件 | 得分 |
|------|---------|------|
| **L1趋势** | MA20在MA40同侧 + 价格在MA40同侧 | 3/3 |
|  | MA20在MA40同侧，但价格在MA40另一侧 | 1/3 |
|  | MA20与MA40反向 | 0/3 |
| **L2资金** | 3日OI轨迹与价格完全同向 | 3/3 |
|  | OI增但价格不涨，或OI减但价格不跌 | 1/3 |
|  | OI与价格完全反向 | 0/3 |
| **L3结构** | 突破前10日/20日高低点 | 3/3 |
|  | 触及结构位但未突破（上下影线） | 1/3 |
|  | 在结构位内震荡 | 0/3 |
| **L4风控** | 止损距离 ≤ 1.0×ATR | 3/3 |
|  | 止损距离 1.0-2.0×ATR | 2/3 |
|  | 止损距离 2.0-3.0×ATR | 1/3 |
|  | 止损距离 > 3.0×ATR（不划算） | 0/3 |

> ⚠️ **L4阈值实现坑点**：L4阈值为**四档**（≤1.0 / 1.0-2.0 / 2.0-3.0 / >3.0），禁止在代码中增加中间档位（如≤1.5得2.5分、≤2.5得1.5分）。中间档位会导致30+品种L4虚高0.5分，总分系统性偏高。实现后必须grep本skill原文交叉验证阈值边界（1.0 / 2.0 / 3.0）。

**置信度计算**：

```
框架置信度分 = 四维总分 / 12 × 100%
（例：四维得 9/12 → 框架置信度分 = 9÷12×100% = 75%）

数据可用性系数 = 见下方置信度体系表格
（例：席位+OI双确认 → 系数 = 1.0）

最终置信度 = 框架置信度分 × 数据可用性系数
（例：75% × 1.0 = 75%，即"高"置信度）
```

**四维评分决策阈值**（由品种特性决定，以下为示例）：

| 四维总分 | 含义 | 决策 |
|------|------|------|
| 总分 ≥ 总分上限×75% | 四维全共振 | 可重仓（仓位系数 1.0） |
| 总分 ≥ 总分上限×58% | 三维确认 | 标准仓位（系数 0.8） |
| 总分 ≥ 总分上限×42% | 两维确认 | 轻仓/观望（系数 0.5） |
| 总分 < 总分上限×42% | 信号弱 | 不做 |

> 示例基于12分总分。高波动品种（如原油）可上调阈值，低波动品种（如贵金属）可下调。

**⚠️ 四维降级路径（数据缺失时的决策规则）**：

四维评分的意义是"有多少维度的数据支持你的假设"，而不是"分数越高越正确"。当某些维度数据不可用时，降级使用，不得强制凑数：

| 缺失维度 | 降级操作 | 评分处理 |
|---------|---------|---------|
| 席位数据不可用 | 使用 L2 OI 轨迹替代验证 | 该维度标记为"降级"，总分上限从12→9 |
| OI 数据延迟超过2天 | 仅用 L1 趋势 + L3 结构判断 | 总分上限从12→6 |
| 盘中实时数据缺失 | 使用昨收价位判断，不强行做盘中决策 | 结论置信度降一档（高→中→低） |
| 快讯数据不可用 | 以昨收价为基础，禁止使用盘中加速判断 | 明确标注"无盘中数据，结论仅供参考" |

**7/12 阈值的真实含义**：这不是"及格线"，而是"有超过一半的可用维度支持你的假设"。如果只有2维可用，7/12 换算成可用维度的比例是 7/12 ÷ (可用维数/4)。**当可用维度不足时，重新计算比例，而不是死守7分。**

> 核心原则：数据不足时，结论的置信度必须降档。降档后的"中置信度"比虚高的"高置信度"更可靠。

IF 三问假设已形成
AND 四维评分 ≥ 7/12
AND 有明确的止损位
AND 席位（若可用则验证，不可用则降档后入场）
AND 已确认各数据源的时间戳（昨收/盘中/席位T-1）
THEN → 入场
```

### 止损规则

| 信号 | 操作 |
|------|------|
| 价格反向突破结构位 | 止损 |
| OI突然大降（>10%）| 先减仓，确认后再处理 |
| 新信号反向（同一品种）| 先减仓，不反手 |
| 席位净持仓突变 | 重新三问穿透 |
| **持仓量降 + 盘中加速**（快讯跌幅>>昨收跌幅）| 谨慎，可能是流动性踩踏 |

### 盘中应对

| 信号 | 解读 | 操作 |
|------|------|------|
| OI增 + 价格不涨 | 空头建仓/多头护盘 | 观察 |
| OI减 + 价格不跌 | 多头平仓/空头护盘 | 观察 |
| OI增 + 价格涨 | 多头建仓 | 持多/跟多 |
| OI减 + 价格跌 | 空头平仓 | 不追 |
| **快讯盘中跌幅 >> 昨收跌幅（×3倍+）** | 行情加速，不逆势 | 持空/加空，不抄底 |

### 仓位管理

```
单笔仓位上限 = 账户净值 × 2%
品种持仓上限 = 账户净值 × 5%
总持仓上限 = 账户净值 × 15%

仓位计算：
  实际手数 = 账户净值 × 仓位系数 / (价格 × 每手吨数 × 保证金率)

**典型品种保证金率参考**（akshare futures_zh_daily_sina 返回字段含 `trade_date`/`vol`/`hold`，保证金率查期货公司官网或经纪商数据）：

| 品种 | 合约 | 保证金率（主力合约） |
|------|------|------------------|
| 螺纹钢 RB | RB2609 | ~9% |
| 铁矿石 I | I2609 | ~12% |
| 焦煤 JM | JM2609 | ~12% |
| 焦炭 J | J2609 | ~12% |
| 沪金 AU | AU2608 | ~8% |
| 沪银 AG | AG2608 | ~10% |
| 沪铜 CU | CU2607 | ~7% |
| 橡胶 RU | RU2609 | ~10% |
| PTA TA | TA2609 | ~8% |

**ATR止损参数N参考值**（由品种波动率决定，使用者可根据历史ATR自设）：

| 波动率 | 品种举例 | 推荐N |
|--------|---------|------|
| 低波动 | 黄金、白银 | 1.5 |
| 中波动 | 螺纹钢、焦煤 | 1.5–2.0 |
| 高波动 | 原油、橡胶、铁矿石 | 2.0–2.5 |
```

| 仓位系数 | 来源 | 说明 |
|---------|------|------|
| 1.0 | 四维评分 ≥ 9/12 | 可重仓 |
| 0.8 | 四维评分 7-8/12 | 标准仓位 |
| 0.5 | 四维评分 5-6/12 | 轻仓 |
| 0 | 四维评分 < 5/12 | 不做 |

### 主动退出规则

在以下条件满足时，**主动平仓**（不等止损位触发）：

| 条件 | 触发操作 | 理由 |
|------|---------|------|
| 浮盈达到入场价的 **3×止损幅度** | 全部平仓 | 盈亏比到位，锁定利润 |
| 价格在目标价位遇到**强阻力**（OI增+价格不涨）| 先减仓50% | 可能有空头护盘 |
| 持仓超过 **3个交易日** 且无浮盈 | 全部平仓 | 时间成本过高 |
| 浮盈回撤超过 **50%**（从最高点）| 全部平仓 | 趋势可能反转 |
| 归因驱动因素消失（新闻反转/政策叫停）| 全部平仓 | 持有逻辑已变 |

## 席位数据缺失时的降级路径

### 降级判断矩阵

| 品种类型 | 席位可用时 | 席位缺失时 | 决策 |
|---------|-----------|-----------|------|
| OI同向确认 + 有明确驱动因素 | 全置信度 | 0.7-0.8 | 可分析，置信度降一档 |
| OI同向确认 + 无明显驱动因素 | 全置信度 | 0.5-0.7 | 可分析但降档，优先选有归因的品种 |
| OI异向（OI增但价格涨，或相反）| 参考席位判断 | 0.3-0.5 | 谨慎，缺席位难以判断多空谁赢 |
| 席位+OI均缺失 + 无归因 | — | 0.2-0.3 | 不建议独立分析 |

### 必须放弃分析的情况

当同时满足以下条件时，**不建议独立入场**：
1. 席位数据不可用
2. OI信号与价格信号矛盾（无法判断多空意图）
3. 没有任何可归因的驱动因素

### 替代验证手段（席位缺失时）

| 原席位功能 | 替代数据 | 替代判断 |

| 验证主力实力 | 仓单变化 | 产业客户行为（若有仓单数据） |

### 框架有效性指标

| 指标 | 计算方式 | 更新频率 |
|------|---------|---------|
| 胜率 | 盈利交易数/总交易数 | 每10笔更新 |
| 平均盈利 | 盈利总额/盈利笔数 | 每10笔更新 |
| 平均亏损 | 亏损总额/亏损笔数 | 每10笔更新 |
| 盈亏比 | 平均盈利/平均亏损 | 每10笔更新 |
| 信号失效特征 | 失效交易共性描述 | 每20笔更新 |

### 信号特征库

按品种归档：
- 什么情况下该品种信号胜率高？
- 什么情况下该品种信号经常失效？
- 失效前兆是什么？
- 加速信号（盘中>>昨收）在该品种上的胜率？

### 复盘结论落地机制

复盘后，**信号特征库的结论必须反馈到下次分析**，具体规则：

| 复盘结论 | 落地操作 |
|---------|---------|
| 某品种"OI增+价格涨"信号历史胜率 > 70% | 下次遇到同样信号，提高L2资金维度得分上限 |
| 某品种"触及结构位但不突破"经常失效 | 下次遇到结构位，评分降一档 |
| 归因强的信号胜率显著高于归因弱的 | 提高归因置信度权重 |
| 加速信号（盘中>>昨收×3）有效预测趋势延续 | 下次遇到加速信号，提高持仓决心 |
| 连续3次止损同类型信号 | 暂停该类型信号，复查框架是否失效 |

**复盘→框架迭代的闭环路径**：

```
每10笔交易 → 统计胜率 → 更新置信度系数基准
每20笔交易 → 分析失效特征 → 调整四维权重或评分规则
每50笔交易 → 全面复盘 → 决定是否修订框架本身
```
