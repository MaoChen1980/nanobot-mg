# MGA — Market Game Analysis

## 核心理念

**穿透现象，看清本质。**

价格/量/OI/席位都是现象。MGA的目标是通过这些现象，还原背后多空博弈方的动机、实力和意图。

**三问穿透法是MGA的灵魂**（必须先于任何工具调用）：
1. **谁在打谁** — 多空身份、动机、成本位
2. **打在哪个价位** — 关键支撑/阻力在哪里
3. **谁能打赢** — 实力对比，席位+OI+仓单

## 三问穿透法（详细版）

### Q1: 谁在打谁

**穿透什么**：多空双方的身份、动机、成本位

| 信号 | 穿透结论 |
|------|---------|
| 席位净空 + OI增 | 空头主力建仓，席位数据验证 |
| 席位净多 + OI减 | 多头止盈，非空头打压 |
| OI增 + 价格涨 | 多头建仓 |
| OI减 + 价格跌 | 空头平仓（非新空进入）|
| 仓单增 + 价格跌 | 产业套保卖盘，压制价格 |
| 仓单减 + 价格涨 | 产业买保，支撑价格 |

**席位数据优先级**（席位是验证"谁"的核心）：
- 净空集中度高 → 空头主力明确
- 净多集中度高 → 多头主力明确
- 席位分散 → 散户行情，非主力行为

### Q2: 打在哪个价位

**穿透什么**：关键支撑/阻力在哪里，多空的成本位和目标位

| 价位类型 | 定义 | 意义 |
|---------|------|------|
| 结构支撑 | 10日/20日/40日低点 | 多头防线 |
| 结构阻力 | 10日/20日/40日高点 | 空头防线 |
| 整数关口 | 15000/14000/10000等 | 心理位置 |
| 前高/前低 | 历史极值 | 趋势确认位 |
| 持仓密集区 | OI堆积价位 | 止损密集区 |

**关键判断**：价格跌破支撑后，如果OI不增，说明空头没认真打，可能是假跌破。

### Q3: 谁能打赢

**穿透什么**：多空双方实力对比

| 实力指标 | 强空头 | 强多头 |
|---------|-------|-------|
| 席位 | 净空大 + 席位集中 | 净多大 + 席位集中 |
| OI | 持续增仓 | 持续增仓 |
| 仓单 | 仓单增（产业卖保） | 仓单减（产业买保） |
| 基差 | 贴水加深 | 升水扩大 |
| 价格 | 跌破结构位不反弹 | 突破结构位不回落 |

## 四维确认框架

四维是验证三问的数据工具，不是目的。

| 维度 | 数据来源 | 通过条件 |
|------|---------|---------|
| **L1趋势** | MA20/MA40方向 | 价格在MA40同侧，MA20在MA40同向 |
| **L2资金** | 3日OI轨迹 | OI和价格同向变化 |
| **L3结构** | 10日/20日高低点 | 突破/跌破前高前低 |
| **L4风控** | ATR止损 | 止损=入场价±N×ATR（N由品种特性决定，参考值1.5） |

**四维评分计算规则**：

每维满分 3 分，总分 12 分。

| 维度 | 通过条件 | 得分 |
|------|---------|------|
| **L1趋势** | MA20在MA40同侧 + 价格在MA40同侧 | 3/3 |
|  | MA20在MA40同侧，但价格在MA40另一侧 | 1/3 |
|  | MA20与MA40反向 | 0/3 |
| **L2资金** | 3日OI轨迹与价格完全同向 | 3/3 |
|  | OI增但价格不涨，或OI减但价格不跌 | 1/3 |
|  | OI与价格完全反向 | 0/3 |
| **L3结构** | 突破前10日/20日高低点 | 3/3 |
|  | 触及结构位但未突破（上下影线） | 1/3 |
|  | 在结构位内震荡 | 0/3 |
| **L4风控** | 止损距离 ≤ 1.0×ATR | 3/3 |
|  | 止损距离 1.0-2.0×ATR | 2/3 |
|  | 止损距离 2.0-3.0×ATR | 1/3 |
|  | 止损距离 > 3.0×ATR（不划算） | 0/3 |

**置信度计算**：

```
框架置信度分 = 四维总分 / 12 × 100%
数据可用性系数 = 见下方置信度体系表格
最终置信度 = 框架置信度分 × 数据可用性系数
```

**四维评分决策阈值**（由品种特性决定，以下为示例）：

| 四维总分 | 含义 | 决策 |
|------|------|------|
| 总分 ≥ 总分上限×75% | 四维全共振 | 可重仓（仓位系数 1.0） |
| 总分 ≥ 总分上限×58% | 三维确认 | 标准仓位（系数 0.8） |
| 总分 ≥ 总分上限×42% | 两维确认 | 轻仓/观望（系数 0.5） |
| 总分 < 总分上限×42% | 信号弱 | 不做 |

> 示例基于12分总分。高波动品种（如原油）可上调阈值，低波动品种（如贵金属）可下调。

## 数据清单

### L1 现象层（基础数据）

| 数据 | 说明 | 备注 |
|------|------|------|
| 日线OHLCV | 包含开高低收和持仓量 | 数据源：任意可靠行情源 |
| 实时价格/OI | 当前价格和持仓量快照 | 数据源：任意可靠行情源 |
| 5分钟K线 | 分钟级OHLCV，可组装日线 | 数据源：任意可靠行情源 |
| 成交量 | 当日成交总量 | 数据源：任意可靠行情源 |

### L2 资金层（多空身份）

| 数据 | 说明 | 状态 |
|------|------|------|
| 席位排名/净持仓 | 交易所公布的多空席位排名 | ⚠️ 接口不稳定，按降级路径处理 |
| 仓单数量 | 交易所/期货公司公布的仓单注册单数 | ⚠️ 未探测 |
| 基差 | 现货价格 - 期货价格 | ✅ 可计算 |

### L3 驱动层（归因数据）

| 数据 | 说明 | 状态 |
|------|------|------|
| 商品期货快讯 | 商品期货专属快讯 | ✅ 可用，实时更新 |
| 宏观/市场动态 | 宏观/市场动态 | ✅ 可用 |
| 突发/政策新闻 | 突发事件/政策原文 | ✅ 可用，实时性差 |
| 隐含波动率 | 期权接口 | ⚠️ 未探测 |
| 宏观数据 | 经济指标 | 部分可用 |

## 决策规则

### 入场规则

```
IF 三问假设已形成
AND 四维评分 ≥ 7/12
AND 有明确的止损位
AND 席位（若可用则验证，不可用则降档后入场）
THEN → 入场
```

### 止损规则

| 信号 | 操作 |
|------|------|
| 价格反向突破结构位 | 止损 |
| OI突然大降（>10%）| 先减仓，确认后再处理 |
| 新信号反向（同一品种）| 先减仓，不反手 |
| 席位净持仓突变 | 重新三问穿透 |

### 盘中应对

| 信号 | 解读 | 操作 |
|------|------|------|
| OI增 + 价格不涨 | 空头建仓/多头护盘 | 观察 |
| OI减 + 价格不跌 | 多头平仓/空头护盘 | 观察 |
| OI增 + 价格涨 | 多头建仓 | 持多/跟多 |
| OI减 + 价格跌 | 空头平仓 | 不追 |

### 仓位管理

```
单笔仓位上限 = 账户净值 × 2%
品种持仓上限 = 账户净值 × 5%
总持仓上限 = 账户净值 × 15%

仓位计算：
  实际手数 = 账户净值 × 仓位系数 / (价格 × 每手吨数 × 保证金率)
```

| 仓位系数 | 来源 | 说明 |
|---------|------|------|
| 1.0 | 四维评分 ≥ 9/12 | 可重仓 |
| 0.8 | 四维评分 7-8/12 | 标准仓位 |
| 0.5 | 四维评分 5-6/12 | 轻仓 |
| 0 | 四维评分 < 5/12 | 不做 |

### 主动退出规则

在以下条件满足时，**主动平仓**（不等止损位触发）：

| 条件 | 触发操作 | 理由 |
|------|---------|------|
| 浮盈达到入场价的 **3×止损幅度** | 全部平仓 | 盈亏比到位，锁定利润 |
| 价格在目标价位遇到**强阻力**（OI增+价格不涨）| 先减仓50% | 可能有空头护盘 |
| 持仓超过 **3个交易日** 且无浮盈 | 全部平仓 | 时间成本过高 |
| 浮盈回撤超过 **50%**（从最高点）| 全部平仓 | 趋势可能反转 |
| 归因驱动因素消失（新闻反转/政策叫停）| 全部平仓 | 持有逻辑已变 |

## 置信度体系

| 数据可用性 | 置信度系数 |
|-----------|-----------|
| 席位+OI+归因三确认 | 1.0 |
| 席位+OI双确认 | 1.0 |
| 仅OI+归因确认（席位缺失）| 0.8 |
| 仅OI确认（席位+归因缺失）| 0.7 |
| 仅席位+归因确认（OI模糊）| 0.6 |
| 席位+OI均缺失 | 0.3 |

**最终置信度 = 框架置信度 × 数据可用性系数**

## 席位数据缺失时的降级路径

### 降级判断矩阵

| 品种类型 | 席位可用时 | 席位缺失时 | 决策 |
|---------|-----------|-----------|------|
| OI同向确认 + 有明确驱动因素 | 全置信度 | 0.7-0.8 | 可分析，置信度降一档 |
| OI同向确认 + 无明显驱动因素 | 全置信度 | 0.5-0.7 | 可分析但降档，优先选有归因的品种 |
| OI异向（OI增但价格涨，或相反）| 参考席位判断 | 0.3-0.5 | 谨慎，缺席位难以判断多空谁赢 |
| 席位+OI均缺失 + 无归因 | — | 0.2-0.3 | 不建议独立分析 |

### 必须放弃分析的情况

当同时满足以下条件时，**不建议独立入场**：
1. 席位数据不可用
2. OI信号与价格信号矛盾（无法判断多空意图）
3. 没有任何可归因的驱动因素

### 替代验证手段（席位缺失时）

| 原席位功能 | 替代数据 | 替代判断 |
|-----------|---------|---------|
| 验证"谁在打" | 期权Put/Call比 | 期权结构隐含多空预期 |
| 验证主力实力 | 仓单变化 | 产业客户行为（若有仓单数据） |
| 验证资金意图 | 实时OI趋势 | 盘中OI变化方向 |

## 复盘验证制度

### 信号记录模板

每笔交易/信号后，记录以下字段：

```
| 日期 | 品种 | 方向 | 入场价 | 止损价 | 出场价/结果 |
|------|------|------|--------|--------|------------|
|      |      |      |        |        |            |

四维评分：__/12
归因结论：[有/无] 驱动因素：___
席位层：___（可用/缺失）
置信度：___
结果：___（盈利/止损/持稳/后悔）
复盘备注：___
```

### 框架有效性指标

| 指标 | 计算方式 | 更新频率 |
|------|---------|---------|
| 胜率 | 盈利交易数/总交易数 | 每10笔更新 |
| 平均盈利 | 盈利总额/盈利笔数 | 每10笔更新 |
| 平均亏损 | 亏损总额/亏损笔数 | 每10笔更新 |
| 盈亏比 | 平均盈利/平均亏损 | 每10笔更新 |
| 信号失效特征 | 失效交易共性描述 | 每20笔更新 |

### 信号特征库

按品种归档：
- 什么情况下该品种信号胜率高？
- 什么情况下该品种信号经常失效？
- 失效前兆是什么？

### 复盘结论落地机制

复盘后，**信号特征库的结论必须反馈到下次分析**，具体规则：

| 复盘结论 | 落地操作 |
|---------|---------|
| 某品种"OI增+价格涨"信号历史胜率 > 70% | 下次遇到同样信号，提高L2资金维度得分上限 |
| 某品种"触及结构位但不突破"经常失效 | 下次遇到结构位，评分降一档 |
| 归因强的信号胜率显著高于归因弱的 | 提高归因置信度权重 |
| 连续3次止损同类型信号 | 暂停该类型信号，复查框架是否失效 |

**复盘→框架迭代的闭环路径**：

```
每10笔交易 → 统计胜率 → 更新置信度系数基准
每20笔交易 → 分析失效特征 → 调整四维权重或评分规则
每50笔交易 → 全面复盘 → 决定是否修订框架本身
```
