# v3.x → v4.0 框架重构记录

> 时间：2026-07-18
> 背景：用户认为原 skill 内容过多、逻辑优先级颠倒，触发全面重构

---

## 重构的触发点

用户原话："这个 skill 好多内容是错误的，没有意义的。比如四维确认框架，它到底在做什么有什么用？"

核心问题：框架的优先级颠倒了——应该是"因果优先，量价印证"，而不是"量价打分验证假设"。

---

## 重构前 vs 重构后对比

| 维度 | v3.x（旧） | v4.0（新） |
|------|-----------|------------|
| **起点** | 先有三问假设 | 先有数据异常/趋势 |
| **核心逻辑** | 因果 ← 量价打分验证 | 因果优先 → 量价印证 |
| **判断工具** | 四维打分（12分制） | 三问（什么资金/想干嘛/能打赢） |
| **入场门槛** | 无明确门槛 | 识别不了就不做 |
| **退出信号** | 止损位 | 因果+量价背离时撤退 |
| **反向分析** | 独立大章节 | 内嵌于 Step 4（不匹配触发） |
| **Pitfalls** | 大段内容 | 删除（后续 review 验证） |
| **品种知识** | 产业链描述 | 定价力量+赢的条件+量价规律 |

## 重构过程中的关键修正

### 修正1：Step 0 扫描不是"找异常"
- 初始理解：扫描是为了找异常
- 正确理解：正常趋势也需要验证，扫描是每天必做
- 触发结果二分为：有异常/无异常但趋势在延续 → 都进入 Step 1

### 修正2：反向分析的位置
- 初始理解：反向分析是独立章节
- 正确理解：反向分析内嵌于 Step 4，触发条件是"量价和因果不匹配"

### 修正3：全市场扫描的触发
- 初始理解：感受到短期资金活跃时启用
- 正确理解：全市场扫描本身就是发现"鲨鱼来了"的方式，而不是因为鲨鱼来了才扫描

### 修正4："识别不了就不做"
- 不是"分析框架得出结论后说识别不了"
- 而是"入场前的过滤器，识别不了直接不做"

---

## 用户核心思想（v4.0 基础）

1. **定价决定性力量是核心**：市场价格是由某股决定性力量推动的，识别这股力量是分析的核心
2. **资金可以扭曲因果**：在小品种上尤其明显，量价关系可能是被制造的
3. **大资金流动性困境**：大资金建仓/平仓需要流动性，因此必须制造成交量，散户看到的趋势可能是诱饵
4. **退出信号**：不是等止损位，而是等因果和量价开始背离
5. **品种知识库**：记的是规律，不是精确公式；每次因素都不一样，只能积累经验规律

## 品种知识库重构

### 旧格式
- 品种简介
- 产业链结构
- 现货定价逻辑

### 新格式
- 定价决定性力量在哪一方
- 什么条件下这方会赢
- 历史量价表现规律
- 容易被操纵的特征
