# 数据采集与核验

## 采集顺序

| 层级 | 要回答的问题 | 首选来源 | 使用规则 |
|---|---|---|---|
| 盘中候选 | 哪些高流动性品种出现异常波动？ | `market-scan.mjs` 的连续主力报价 | 仅筛选候选；记录报价时间，不把连续主力当成下单合约。 |
| 官方行情 | 涨跌、成交、持仓、结算是否真实？ | [上期所](https://www.shfe.com.cn/)、[大商所](https://www.dce.com.cn/)、[郑商所](https://www.czce.com.cn/)、[广期所](https://www.gfex.com.cn/)、[中金所](https://www.cffex.com.cn/) | 用同一合约的官方结算价和持仓量；盘后下载或查询。 |
| 资金结构 | 谁在加多、加空，集中度是否提高？ | 各交易所“成交持仓排名/前二十会员” | 计算前二十多、空持仓净变化；没有排名就写“无法确认席位”。 |
| 现货与交割 | 供应是否真的收紧或放松？ | 交易所仓单日报、交割库数据、行业协会/现货机构 | 记录库存口径、日期和环比；不混用不同机构口径。 |
| 宏观与海外 | 海外定价锚是否同向？ | [EIA](https://www.eia.gov/)、[USDA](https://www.usda.gov/)、[OPEC](https://www.opec.org/)、[国家统计局](https://www.stats.gov.cn/)、[海关总署](https://www.customs.gov.cn/) | 只用与品种相关的发布；标注发布日期和下次发布时间。 |
| 事件 | 异动有没有可验证的外部原因？ | 交易所公告、政府/监管公告、公司公告、上述原始发布 | 新闻只能解释，不能单独证明资金方向。 |

## 指标口径

- **盘中时效**：`market-scan.mjs` 只把抓取前 15 分钟内的报价纳入候选；旧报价保留在 JSON 原始结果中，但不得用于当前开仓。
- **涨跌幅**：`(最新价 / 上日官方结算价 - 1) × 100%`。盘中报价的昨结字段用于筛选，盘后必须改用官方结算价复核。
- **增仓**：`当日官方持仓量 - 前一交易日官方持仓量`，且必须是同一合约。
- **V/OI**：连续主力报价中的成交量/持仓量仅作盘中热度代理；收盘后用官方数据复核，不能直接代表资金净流入。
- **席位净变化**：前二十会员多头持仓变化减前二十会员空头持仓变化；它反映公开会员口径，不等同于全部市场资金。
- **成交额**：必须使用精确合约乘数。无法取得乘数时，不对跨品种成交额排序。

## 硬性核验规则

1. 报价时间不在当前交易时段、交易日期不一致，或连续主力与实际合约映射不清：只观察。
2. 盘中信号与盘后官方结算/持仓不一致：以官方数据为准，并撤销该信号。
3. 没有席位、库存或事件中至少一项来解释价格：不把行情描述为“主力建仓”。
4. 交易所未披露某项数据时，明确写“数据不可得”；不要用接口猜测替代。
5. 保存每轮分析的来源链接、抓取时间、合约代码和口径，避免把不同日期的数据拼在一起。

## 脚本用法

```powershell
node references/market-scan.mjs
node references/market-scan.mjs SC0 AU0 AG0
node references/market-scan.mjs --json
```

默认池覆盖能源、贵金属、有色、黑色、油脂油料、化工、软商品、养殖和碳酸锂的高流动性连续主力。中金所品种不纳入该报价源，须直接查中金所行情与成交持仓排名。
