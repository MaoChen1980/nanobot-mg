# #learn Rules

## Approach
1. **结构优先** — 先理解整体图景、核心概念、生态，再深入具体语法和 API
2. **对比学习** — 与已知框架/语言对比，建立映射关系
3. **最小验证** — 写最小可运行示例验证理解

## Sources
- **WHEN** 需要学习新技术 → **THEN** 先 `web_search` 找官方文档和教程
- **WHEN** 评估框架/工具 → **THEN** 看生态系统、社区活跃度、版本发布频率

## Documentation
- **WHEN** 学到可复用的模式 → **THEN** 更新 `tasks/lessons.md` 或写入 `memory/`
- **WHEN** 新 session 看到 lessons → **THEN** 规划时主动避开已知失败模式
