评估下面的消息内容，判断是否适合直接发送给用户。

只输出纯 JSON，不要使用 markdown 代码块。

字段说明：
- status: "ok"（正常，可直接发送）或 "issues"（存在问题）
- issues: 问题列表，每个问题包含描述
- summary: 一句话总结

检查项：
1. 内容是否为空或近乎为空
2. 是否包含不应出现在用户视角的调试/评估标记（如 [assess]、[/assess]、[tool_summary]、[/tool_summary]、<!-- no-assess -->）
3. 内容是否不完整或明显残缺（如以"待续"、"未完"等结尾，或明显缺少关键部分）
4. 内容是否包含明显的事实性错误（与对话上下文矛盾）
5. 是否存在安全性问题（如泄露敏感信息）

示例输出：
{"status": "ok", "issues": [], "summary": "内容完整，可直接发送"}

示例输出（有问题）：
{"status": "issues", "issues": ["包含 assess 标记", "内容以'未完待续'结尾，不完整"], "summary": "包含调试标记且内容不完整"}

消息内容：
{{ content }}

{% if context %}
最近对话上下文：
{{ context }}
{% endif %}

约束：
- 检查依据必须是消息内容本身，不要过度推断
- 事实性错误需有对话上下文中的明确依据，不捕风捉影
- 无明确问题则 status 为 "ok"，issues 为空数组
